# 工具脚本使用手册

> 位置：`xlsx/` 目录下的辅助脚本 + `colorize_md.py`
> 用途：内容生成、格式检查、批量修复、导航维护

---

## 一、核心数据文件

### `Linux知识体系.xlsx`
- **位置**：`xlsx/Linux知识体系.xlsx`
- **用途**：全部知识体系的唯一数据源，8大模块的知识总纲
- **sheet列表**：
  - `总纲目录` — 8大模块的目录结构
  - `硬件层`、`Bootloader`、`Linux内核`、`系统构建`、`驱动开发`、`调试与优化`、`应用层`、`总线协议` — 各模块详细内容
- **使用方式**：所有内容写作前必读，确认知识点范围和难度等级

---

## 二、内容生成脚本

### `generate_from_xlsx.py` / `generate_v2.py`
- **用途**：从xlsx自动生成Markdown内容
- **用法**：`python xlsx/generate_from_xlsx.py`
- **输出**：按模块生成.md文件到对应目录
- **注意**：生成后需要人工审核格式（四色标记、BIEM徽章等）

### `prep_task_sheets.py`
- **用途**：预处理xlsx，提取任务清单
- **用法**：`python xlsx/prep_task_sheets.py`
- **输出**：`xlsx/prep_extract/task_sheets.json` 和 `.md`

## 三、格式检查脚本

### `check_*.py` 系列（批量检查工具）

| 脚本名 | 用途 | 用法 |
|--------|------|------|
| `check_06.py` | 检查06号规则（表格内span标签） | `python xlsx/check_06.py <目录>` |
| `check_columns.py` | 检查表格列对齐 | `python xlsx/check_columns.py <目录>` |
| `check_files.py` | 检查文件命名规范 | `python xlsx/check_files.py <目录>` |
| `check_format.py` | 综合格式检查（段落长度、换行符等） | `python xlsx/check_format.py <目录>` |
| `check_generated.py` | 检查自动生成内容的完整性 | `python xlsx/check_generated.py <目录>` |
| `check_headings.py` | 检查标题层级规范 | `python xlsx/check_headings.py <目录>` |
| `check_spans.py` | 检查span标签使用合规性 | `python xlsx/check_spans.py <目录>` |
| `check_tables.py` | 检查表格格式 | `python xlsx/check_tables.py <目录>` |

**通用用法**：
```bash
cd D:\你的项目目录
python xlsx/check_format.py "docs/03-Linux内核与驱动/"
```

**建议检查顺序**：
1. `check_files.py` — 先确认文件名规范
2. `check_headings.py` — 确认标题层级
3. `check_spans.py` — 确认颜色标签合规
4. `check_tables.py` — 确认表格无span标签
5. `check_format.py` — 综合检查

## 四、批量修复脚本

### `fix_*.py` 系列（自动修复工具）

| 脚本名 | 修复内容 | 用法 |
|--------|----------|------|
| `fix_all.py` | 一键修复所有已知问题 | `python xlsx/fix_all.py <目录>` |
| `fix_all_files.py` | 逐文件修复 | `python xlsx/fix_all_files.py <目录>` |
| `fix_aggressive.py` | 激进修复模式（强力清理） | `python xlsx/fix_aggressive.py <目录>` |
| `fix_br.py` | 修复换行符问题（`<br>`标签） | `python xlsx/fix_br.py <目录>` |
| `fix_spans_comprehensive.py` | 综合修复span标签 | `python xlsx/fix_spans_comprehensive.py <目录>` |
| `fix_tables.py` | 修复表格格式 | `python xlsx/fix_tables.py <目录>` |
| `fix_tables_blank.py` | 修复表格空白单元格 | `python xlsx/fix_tables_blank.py <目录>` |

**推荐用法**：
```bash
# 先用fix_all一键修复
python xlsx/fix_all.py "docs/"

# 如果还有残留问题，用fix_aggressive深度修复
python xlsx/fix_aggressive.py "docs/03-Linux内核与驱动/"

# 最后跑check验证
python xlsx/check_format.py "docs/"
```

## 五、导航维护脚本

### `compress_nav.py`
- **用途**：压缩mkdocs.yml导航层级，合并过深的子目录
- **用法**：`python xlsx/compress_nav.py`
- **注意**：修改前先备份mkdocs.yml

### `export_structure.py`
- **用途**：导出当前目录结构为JSON/Markdown
- **用法**：`python xlsx/export_structure.py`
- **输出**：当前docs/目录的完整结构清单

### `create_readme.py`
- **用途**：为没有README.md的目录自动生成概览文件
- **用法**：`python xlsx/create_readme.py`
- **建议**：新建模块后先跑这个脚本生成骨架

## 六、四色标记上色脚本

### `colorize_md.py`（项目根目录）
- **位置**：`colorize_md.py`（与mkdocs.yml同级）
- **用途**：批量给Markdown文件自动上色（red/green/blue/orange）
- **核心规则**：
  - `red` — 核心概念词（内核、进程、调度器、中断等）
  - `green` — 技术术语/API（kmalloc、request_irq、I2C、SPI等）
  - `blue` — 关键结论句
  - `orange` — 有序列表项标题

**用法**：
```bash
# 给03模块全部文件上色
cd D:\你的项目目录
python colorize_md.py "docs/03-Linux内核与驱动/"

# 给05模块上色
python colorize_md.py "docs/05-驱动开发/"

# 给所有模块上色
python colorize_md.py "docs/"
```

**注意事项**：
- 上色前先用git备份（或手动备份）
- 脚本会跳过已有颜色的文件
- 上色后务必用 `check_spans.py` 验证
- 暗色模式下颜色会变化（extra.css已适配）

## 九、标准工作流程

### 场景1：新建模块后初始化
```bash
# 1. 从xlsx生成内容骨架
python xlsx/generate_from_xlsx.py

# 2. 为缺失README的目录生成概览
python xlsx/create_readme.py

# 3. 导出目录结构检查
python xlsx/export_structure.py

# 4. 更新mkdocs.yml导航（手动或自动）
```

### 场景2：内容写完后的格式检查
```bash
# 1. 检查文件命名
python xlsx/check_files.py "docs/新模块/"

# 2. 检查标题层级
python xlsx/check_headings.py "docs/新模块/"

# 3. 修复格式问题
python xlsx/fix_all.py "docs/新模块/"

# 4. 检查span标签合规性
python xlsx/check_spans.py "docs/新模块/"

# 5. 检查表格
python xlsx/check_tables.py "docs/新模块/"

# 6. 综合诊断
python xlsx/diagnose.py
```

### 场景3：批量上色（已有内容补颜色）
```bash
# 1. 备份（重要！）
cp -r docs docs_backup

# 2. 运行上色脚本
python colorize_md.py "docs/03-Linux内核与驱动/"

# 3. 验证颜色渲染
python xlsx/check_spans.py "docs/03-Linux内核与驱动/"

# 4. mkdocs serve查看效果
mkdocs serve
```

### 场景4：导航精简
```bash
# 1. 备份mkdocs.yml
cp mkdocs.yml mkdocs.yml.bak

# 2. 压缩导航层级
python xlsx/compress_nav.py

# 3. 验证YAML语法
python -c "import yaml; yaml.safe_load(open('mkdocs.yml'))"

# 4. 重启serve
mkdocs serve
```
