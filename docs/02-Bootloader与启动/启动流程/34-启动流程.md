# 启动流程

> 📊 **本章难度等级：** <span class="badge-i">**中级 (Intermediate)**</span>

---

### <strong>CPU从固定地址（ARM：0xFFFF0000，RISC-V：0x1004）执行第一条指令；SoC内置ROM代码加载引导介质首扇区到SRAM</strong>


### <strong>时钟树配置（PLL锁相环）→ 内存控制器校准（DDR PHY训练）→ 基础外设使能（调试串口/UART0）</strong>


### <strong>根据Boot Mode引脚（eFUSE/GPIO）选择源：NOR Flash/eMMC/SD卡/网络（PXE）。</strong>


### <strong>SPL（U-Boot SPL或ARM TF-A BL1）初始化DDR → 加载完整U-Boot到内存 → 移交执行权。</strong>


### <strong>驱动初始化：存储控制器（eMMC/SD）、网络接口（MAC+PHY）
环境变量加载：bootcmd决定启动路径（本地/网络）
内核加载：将zImage/dtb/initramfs从存储或TFTP服务器拷贝到指定内存地址</strong>


### <strong>自解压程序（decompressor）展开vmlinux → 初始化MMU/异常向量 → 解析设备树（DTB物理地址由U-Boot传入）。</strong>


### <strong>调度器（sched_init）→ 中断控制器（irqchip_init）→ 定时器（time_init）→ 内存管理（mm_init）。</strong>


### <strong>网卡驱动注册：module_init()注册PCIe/USB/Ethernet驱动
网络栈初始化：net_dev_init()创建lo设备 → 协议栈（TCP/IP）初始化
PHY链路训练：phylink_start()触发自协商（100ms~2s）</strong>


### <strong>内核根据root=参数挂载rootfs（initramfs/磁盘镜像）→ 切换根目录 → 启动/sbin/init（PID 1）。</strong>


### <strong>驱动层：udev加载网卡固件（firmware_class）→ 应用MAC地址（ifconfig eth0 hw ether）
配置层：systemd-networkd应用/etc/network/interfaces（静态IP）或启动dhcpcd（DHCP）
服务层：network-online.target触发依赖服务（SSH/NTP）</strong>


### <strong>netif_running()为真（驱动probe完成）→ netif_carrier_ok()为真（物理链路up）→ 生成NETDEV_UP事件。</strong>


### <strong>IP地址分配：ip addr show dev eth0显示有效IPv4/IPv6
路由可用：默认网关可达（route -n）
服务响应：systemctl is-active systemd-networkd返回active</strong>


### <strong>阶段        典型耗时       瓶颈案例
Bootloader  300ms~1.5s    eMMC初始化时钟同步
内核启动    0.8s~3s        并发驱动probe阻塞
网络服务    0.5s~5s        DHCP重试超时</strong>


### <strong>U-Boot：禁用USB/PCIE扫描（CONFIG_DM_USB=n）
内核：异步驱动初始化（driver_async_probe）
用户空间：静态IP替代DHCP，并行启动服务（systemd的After=network-online.target）</strong>
