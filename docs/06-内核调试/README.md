# 06-内核调试与性能优化

> **模块定位**: 内核调试与性能优化是嵌入式 Linux 开发中从"能用"到"好用"的关键跃迁。本模块覆盖从基础日志调试到高级 eBPF 动态追踪、从崩溃分析到实时性优化的完整技术栈。
>
> **难度跨度**: [B] 基础认知 → [I] 中级实践 → [E] 高级深入 → [M] 大师级前沿

## 模块导览

| 章节 | 主题 | 难度 | 核心内容 |
|------|------|------|----------|
| 01 | 内核调试基础认知 | [B] | printk、动态调试、earlyprintk、日志系统 |
| 02 | Oops 与 panic 分析 | [I] | 崩溃消息解读、寄存器分析、调用栈回溯 |
| 03 | kgdb 与 kdb 调试器 | [I→E] | 远程调试架构、断点命令、嵌入式实战 |
| 04 | ftrace 动态追踪 | [E] | function tracer、tracepoint、条件追踪 |
| 05 | eBPF 与 BCC 工具 | [E] | BPF 验证器、BCC/BPFtrace、uprobes/kprobes |
| 06 | perf 与火焰图 | [E] | 事件采样、热点定位、缓存分析、嵌入式裁剪 |
| 07 | Kconfig 与内核裁剪 | [I→E] | 配置语法、依赖关系、最小化策略 |
| 08 | 调度与实时性分析 | [E→M] | 调度延迟、PREEMPT_RT、优先级继承 |
| 09 | 内存调试与 SLUB | [E] | slab 调试、KASAN、内存泄漏、越界检测 |
| 10 | 启动时间优化 | [I→E] | 启动流程、initcall_debug、并行初始化 |
| 11 | 内核性能分析 | [E] | PMU 硬件事件、CPU 缓存、分支预测 |
| 12 | 内核调试历史演进 | [E→M] | 从 printk 到 eBPF、CoreSight、AI 辅助调试 |

## 学习路线建议

- **入门路径**: 01 → 02 → 03 → 07 → 10
- **性能优化路径**: 04 → 05 → 06 → 08 → 11
- **深度调试路径**: 02 → 03 → 09 → 05 → 12

> 💡 **提示**: 每个章节文件开头均标注 BIEM 徽章，表示该章节的难度等级与知识深度。
