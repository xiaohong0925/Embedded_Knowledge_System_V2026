### <strong>内核核心结构体：驱动与内核的“交互桥梁”
Linux内核通过两大核心结构体抽象网络<span class="red">设备驱动</span>的核心逻辑：`net_device`（描述网卡硬件属性与驱动操作）和`sk_buff`（封装数据包的“容器”）。
前者是“网卡的身份证与操作手册”，后者是“数据包的运输车厢”，二者协同完成数据收发的全流程。</strong>

4.4.1 `net_device`：网卡的“全景描述书”

定义：
`net_device`结构体是内核管理网络设备的核心抽象，包含网卡的硬件信息（如MAC地址、MTU）、功能标志（如是否支持广播、组播）、驱动操作集（如收发函数）等，驱动开发的核心工作本质是“正确初始化`net_device`并注册到内核”。<br>

### <strong>数据包收发流程：驱动与内核的“协同链路”</strong>

数据包的“接收”和“发送”是网络<span class="red">设备驱动</span>的核心业务流程，驱动在其中扮演“硬件与协议栈的翻译官”角色
——接收时将硬件电信号转换为`sk_buff`，发送时将`sk_buff`转换为硬件信号。
以下分两条链路详解，结合流程图和驱动函数的交互逻辑，明确各环节的责任边界。<br>

### <strong>驱动注册流程：网卡“上线”的全流程
网络设备驱动的注册流程是“将`net_device`结构体注册到内核<span class="red">网络子系统</span>”的过程，
核心步骤为“分配设备→初始化→注册→注销”，是驱动开发的基础流程，需掌握工业级的实现规范（如错误处理、资源释放）。</strong>

### <strong>硬件基础：DM9000网卡与SOC的协同逻辑</strong>

以太网驱动开发的前提是明确“网卡-总线-SOC”的硬件连接关系，DM9000作为一款经典的10/100Mbps自适应以太网控制器，广泛适配ISA和<span class="green">PCIe</span>总线，其硬件交互逻辑具有代表性，掌握后可迁移至其他网卡型号。

4.7.1 核心硬件连接关系（I级：必懂拓扑）

### <strong>驱动核心：probe函数的实战实现
probe函数是驱动与硬件的“首次握手”，当SOC检测到DM9000网卡（通过<span class="red">设备树</span>匹配或总线枚举）后，会自动调用probe函数完成硬件初始化、资源映射和函数绑定。以下基于Linux 5.10内核，实现工业级probe函数并解析关键步骤。</strong>

4.8.1 probe函数核心流程（I级：步骤拆解）

probe函数的核心任务是“资源初始化+功能绑定”，
需按“硬件资源获取→寄存器映射→硬件复位→核心参数配置→函数绑定”的顺序执行，步骤及对应代码如下：

```c
#include <linux/platform_device.h>
#include <linux/netdevice.h>
#include <linux/etherdevice.h>
#include <linux/io.h>
#include <linux/mdio.h>
#include <linux/phy.h>

// 驱动私有数据：存储硬件资源和状态
struct dm9000_dev {
    void __iomem *io_addr;    // 寄存器基地址（映射后）
    void __iomem *data_addr;  // 数据缓冲区地址（映射后）
    struct net_device *ndev;  // 关联的net_device结构体
    struct phy_device *phydev;// 关联的PHY设备
    int irq;                  // 中断号
};

// probe函数入口
static int dm9000_probe(struct platform_device *pdev) {
    struct dm9000_dev *dev;
    struct net_device *ndev;
    struct resource *res_io, *res_data, *res_irq;
    int ret;

    // 步骤1：分配私有数据和net_device结构体
    dev = devm_kzalloc(&pdev->dev, sizeof(*dev), GFP_KERNEL);
    if (!dev) return -ENOMEM;
    ndev = alloc_netdev(sizeof(struct dm9000_dev), "eth%d", NET_NAME_UNKNOWN, ether_setup);
    if (!ndev) return -ENOMEM;
    dev->ndev = ndev;
    netdev_priv(ndev) = dev;
    platform_set_drvdata(pdev, dev);

    // 步骤2：获取并映射硬件资源（从设备树读取）
    // 2.1 获取寄存器地址资源
    res_io = platform_get_resource_byname(pdev, IORESOURCE_MEM, "dm9000_io");
    res_data = platform_get_resource_byname(pdev, IORESOURCE_MEM, "dm9000_data");
    // 映射寄存器地址（devm_前缀自动释放资源）
    dev->io_addr = devm_ioremap_resource(&pdev->dev, res_io);
    if (IS_ERR(dev->io_addr)) return PTR_ERR(dev->io_addr);
    dev->data_addr = devm_ioremap_resource(&pdev->dev, res_data);
    if (IS_ERR(dev->data_addr)) return PTR_ERR(dev->data_addr);

    // 2.2 获取中断资源
    res_irq = platform_get_resource_byname(pdev, IORESOURCE_IRQ, "dm9000_irq");
    dev->irq = res_irq->start;
    if (dev->irq < 0) return -EINVAL;

    // 步骤3：硬件复位与初始化
    dm9000_soft_reset(dev);  // 软件复位（写入复位命令到控制寄存器）
    msleep(100);             // 等待复位完成

    // 步骤4：MAC地址初始化（优先从设备树读取，其次从网卡EEPROM读取）
    ret = of_property_read_u8_array(pdev->dev.of_node, "local-mac-address", 
                                   ndev->dev_addr, ETH_ALEN);
    if (ret) {
        // 从DM9000 EEPROM读取MAC地址
        dm9000_read_eeprom(dev, 0, ndev->dev_addr, ETH_ALEN);
        if (!is_valid_ether_addr(ndev->dev_addr)) {
            eth_hw_addr_random(ndev);  // 随机生成（调试用）
            dev_warn(&pdev->dev, "use random MAC: %pM\n", ndev->dev_addr);
        }
    }

    // 步骤5：PHY芯片初始化（通过MDIO总线）
    dev->phydev = phy_find_first(dev->ndev);  // 查找PHY设备
    if (!dev->phydev) {
        dev_err(&pdev->dev, "phy not found\n");
        return -ENODEV;
    }
    phy_connect(dev->ndev, dev->phydev, dm9000_phy_handler, 0);  // 绑定PHY处理函数
    phy_start(dev->phydev);  // 启动PHY芯片

    // 步骤6：中断注册（绑定中断处理函数）
    ret = devm_request_irq(&pdev->dev, dev->irq, dm9000_interrupt, 
                          IRQF_SHARED | IRQF_TRIGGER_RISING, "dm9000", ndev);
    if (ret) {
        dev_err(&pdev->dev, "request irq failed: %d\n", ret);
        return ret;
    }

    // 步骤7：绑定net_device_ops操作集
    ndev->netdev_ops = &dm9000_netdev_ops;
    ndev->hard_header_len = ETH_HLEN;
    ndev->mtu = ETH_DATA_LEN;

    // 步骤8：注册net_device到内核
    ret = register_netdev(ndev);
    if (ret) {
        dev_err(&pdev->dev, "register netdev failed: %d\n", ret);
        free_netdev(ndev);
        return ret;
    }

    dev_info(&pdev->dev, "probe success: MAC=%pM, IRQ=%d, PHY=%s\n", 
             ndev->dev_addr, dev->irq, dev->phydev->name);
    return 0;
}
    
```<br>

4.9.1 基础调试环境搭建（I级：必做步骤）

调试前需确保内核、驱动、文件系统满足以下条件，避免因环境问题导致调试失败：
- *内核配置：
启用DM9000驱动和相关网络功能，配置路径及选项：
        `

Device Drivers > Network device support > PHY Device support > Generic PHY support
# 启用网络工具（ifconfig、ping、ethtool）
Networking support > Networking tools > [*] ifconfig support [*] ping support
        `

- 驱动加载：
若驱动编译为模块，通过以下命令加载并查看加载状态：
        `
# 加载驱动模块
<span class="green">insmod</span> dm9000.ko

- 硬件连接：
将DM9000网卡RJ45接口连接至路由器或PC（通过交叉线或直连线，现代网卡支持自动翻转），确保链路指示灯（Link/Act）常亮或闪烁。<br>

    };
};
