# 4.2.7 默认配置模板：defconfig的妙用

## 本节导读

上一节我们学会了读懂 `.config` 文件的语法，但面对内核中上万个选项，从零开始逐个勾选是不现实的。本节介绍内核社区提供的"现成模板"——defconfig，教你站在巨人的肩膀上，用一条命令加载经过验证的默认配置，再在此基础上按需微调，省时省力不出错。

## 知识点1：defconfig是什么 [B] ~800字

### 概念：内核社区的" starter 模板"

**defconfig**（default config 的缩写）是内核社区为常见硬件平台预先准备好的配置文件。你可以把它理解为买家具时的"套餐方案"——不是从零挑选每块木板，而是先选一个接近你需求的套餐，再替换一两件单品。

在 Linux 内核源码中，所有 defconfig 文件按 CPU 架构分类存放，路径规则固定为：

```
arch/<架构名>/configs/<平台名>_defconfig
```

例如 ARM64 架构下：

```bash
cd linux-5.15/
ls arch/arm64/configs/
```

输出示例：

```
defconfig                    # ARM64 的通用默认配置
armv8_defconfig              # 纯 64 位 ARMv8 基础配置
rk3399_defconfig             # 瑞芯微 RK3399 开发板
imx8mq_defconfig             # NXP i.MX8M Quad 芯片
tegra_defconfig              # NVIDIA Tegra 系列
...
```

💡 **提示**：不同内核版本的 defconfig 文件数量和名称会有差异。厂商芯片的 defconfig 通常在主线内核合并后才出现，如果你用的是厂商提供的 BSP 内核，可能看到 `rk3588_defconfig`、`imx6ull_defconfig` 等更多选项。

### 加载 defconfig：一条命令生成完整配置

加载 defconfig 的命令格式极其简单：

```bash
cd linux-5.15/
make ARCH=arm64 defconfig          # 加载 ARM64 通用默认配置
# 或者指定具体平台
make ARCH=arm64 rk3399_defconfig   # 加载 RK3399 专用配置
```

这条命令背后发生了什么？构建系统会把 `arch/arm64/configs/rk3399_defconfig` 复制到源码根目录，重命名为 `.config`，然后自动补齐所有缺失的选项（用默认值填充），最终生成一份**完整的、可直接用于编译**的 `.config`。

```mermaid
flowchart LR
    A[执行 make xxx_defconfig] --> B[从 arch/xxx/configs/ 读取模板]
    B --> C[复制到源码根目录作为 .config]
    C --> D[运行 Kconfig 依赖解析]
    D --> E[自动补齐缺失选项为默认值]
    E --> F[生成完整 .config]
    F --> G[可执行 make 编译]
```

[图1：defconfig 加载为完整 .config 的流程图]

⚠️ **陷阱**：`make xxx_defconfig` 会**覆盖**源码根目录下已有的 `.config` 文件，且不会自动备份。如果你之前辛苦配置了一份 `.config`，执行此命令前务必先备份！

### 常见 defconfig 速查表

| 芯片/平台 | 架构 | defconfig 文件名 | 适用场景 |
|:---|:---|:---|:---|
| 通用 ARM64 | arm64 | `defconfig` | 学习、QEMU 虚拟机、无特定硬件 |
| 树莓派 3/4 | arm64 | `bcm2835_defconfig` / `bcm2711_defconfig` | 树莓派开发板 |
| 瑞芯微 RK3399 | arm64 | `rk3399_defconfig` | 友善之臂 NanoPC-T4、Firefly |
| 瑞芯微 RK3588 | arm64 | `rk3588_defconfig` | 高端开发板、AI 推理场景 |
| NXP i.MX8M | arm64 | `imx8mq_defconfig` / `imx8mm_defconfig` | 工业控制、多媒体网关 |
| NVIDIA Jetson | arm64 | `tegra_defconfig` | AI 边缘计算、Jetson 系列 |
| 全志 H6/H616 | arm64 | `sunxi_defconfig` | 电视盒子、低成本单板 |
| x86 通用 PC | x86 | `x86_64_defconfig` / `i386_defconfig` | PC 服务器、虚拟机 |

💡 **提示**：厂商 BSP 内核中，defconfig 文件名可能更具体，例如 `rv1126_defconfig`、`imx6ull_14x14_evk_defconfig`，通常可以在 `arch/arm/configs/`（32位）或 `arch/arm64/configs/`（64位）下找到它们。

## 知识点2：savedefconfig 生成精简配置 [I] ~600字

### 从臃肿到精简：为什么要用 savedefconfig

一份完整的 `.config` 文件通常包含 **5000~15000 行**，但其中绝大部分选项你根本没碰过——它们只是默认配置。直接把这个庞然大物存到 Git 仓库或分享给同事，既浪费空间又难以看清"你究竟改了什么"。

`make savedefconfig` 的作用就是**提取差异**：它把你当前 `.config` 中与默认值不同的选项单独拎出来，生成一个精简的 `defconfig` 文件。

### 操作步骤

```bash
cd linux-5.15/
# 第1步：先加载一个基础配置
make ARCH=arm64 defconfig
# 第2步：用 menuconfig 按需修改
make ARCH=arm64 menuconfig
# 第3步：保存精简差异配置
make ARCH=arm64 savedefconfig
```

执行后，源码根目录会生成一个名为 `defconfig` 的文件（无下划线前缀），体积通常只有几十到几百行：

```bash
$ wc -l .config defconfig
  15234 .config        # 完整配置，1.5万行
    127 defconfig       # 精简差异，127行
```

### 生成的 defconfig 长什么样

```bash
# 示例：savedefconfig 生成的精简文件
cat defconfig
```

输出示例：

```
CONFIG_LOCALVERSION="-custom"
CONFIG_SYSVIPC=y
CONFIG_POSIX_MQUEUE=y
CONFIG_NO_HZ=y
CONFIG_HIGH_RES_TIMERS=y
CONFIG_PREEMPT=y
CONFIG_BPF_SYSCALL=y
CONFIG_EMBEDDED=y
# ... 仅包含被修改过的选项
```

注意这个文件**没有 `CONFIG_` 前缀之外的多余内容**，也没有注释掉的选项。它就是一个"最小修改集"。

### 用精简 defconfig 恢复完整配置

把 `defconfig` 文件拷贝到 `arch/arm64/configs/myboard_defconfig`，之后任何人都可以：

```bash
make ARCH=arm64 myboard_defconfig    # 恢复为你的定制配置
```

⚠️ **陷阱**：`make savedefconfig` 生成的文件默认放在源码根目录且命名为 `defconfig`，容易与 `arch/xxx/configs/defconfig` 混淆。建议生成后立即重命名：`mv defconfig arch/arm64/configs/my_custom_defconfig`。

💡 **提示**：在 CI/CD 自动化编译场景中，`savedefconfig` 是保存"配置快照"的最佳方式。把生成的 defconfig 提交到版本控制，构建流水线中只需执行 `make xxx_defconfig` 即可恢复一致的编译环境。

## 知识点3：从已知好配置出发 [I] ~500字

### 找到你的板子的默认配置

嵌入式开发的首要原则：**不要从零开始**。几乎每一块主流开发板，内核社区或厂商都已经提供了一份经过验证的 defconfig。找到它的方法有三种：

**方法一：在内核源码中搜索**

```bash
# 按芯片型号搜索
cd linux-5.15/
ls arch/arm64/configs/ | grep -i rk3588
ls arch/arm64/configs/ | grep -i imx

# 如果找不到，可能藏在 32 位 ARM 目录
ls arch/arm/configs/ | grep -i imx6
```

**方法二：查阅开发板文档**

正点原子、友善之臂、迅为等开发板厂商，通常会在用户手册中写明推荐使用的 defconfig 名称，例如：

> "编译内核前，请先执行 `make ARCH=arm64 imx8mq_defconfig`"

**方法三：参考厂商 BSP 仓库**

如果主线内核没有你的芯片的 defconfig，去芯片厂商的 GitHub 或 GitLab 仓库查找 BSP 版本内核。例如瑞芯微的 `kernel-5.10` 分支中通常包含 `rk3568_defconfig`、`rk3588s_defconfig` 等。

### 在好配置基础上微调的标准流程

找到默认配置后，推荐的迭代流程如下：

```bash
cd linux-5.15/

# Step 1: 加载已知可用的默认配置
make ARCH=arm64 rk3399_defconfig

# Step 2: 在此基础上微调（增删驱动、调整内核参数）
make ARCH=arm64 menuconfig

# Step 3: 编译验证
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)

# Step 4: 如果工作正常，保存为精简 defconfig 供日后复用
make ARCH=arm64 savedefconfig
mv defconfig arch/arm64/configs/my_rk3399_defconfig
```

🔴 **危险**：不要在未加载任何 defconfig 的情况下直接执行 `make menuconfig`。此时内核构建系统没有任何默认选项，你需要手动回答每一个选项的取值，这在实践中几乎不可能完成。

💡 **提示**：如果你不确定某个驱动是否已启用，可以在加载 defconfig 后用 `grep` 快速检查：

```bash
make ARCH=arm64 rk3399_defconfig
grep CONFIG_USB_STORAGE .config    # 查看 USB 大容量存储是否已启用
grep CONFIG_USB_HID .config        # 查看 USB 鼠标键盘是否已启用
```

## 本节总结

| 概念 | 要点 | 操作 |
|:---|:---|:---|
| defconfig | 按架构存放在 `arch/xxx/configs/` 下的默认配置模板 | `ls arch/arm64/configs/` 查看可用模板 |
| 加载 defconfig | 用 `make xxx_defconfig` 一键生成完整 `.config` | `make ARCH=arm64 rk3399_defconfig` |
| savedefconfig | 提取当前配置与默认的差异，生成精简文件 | `make savedefconfig`，输出为 `defconfig` |
| 保存自定义配置 | 将精简 defconfig 放入 `arch/xxx/configs/` 目录 | `mv defconfig arch/arm64/configs/my_defconfig` |
| 标准工作流 | 加载默认配置 → menuconfig 微调 → 编译验证 → 保存快照 | `make xxx_defconfig && make menuconfig && make` |
| 安全提醒 | `make xxx_defconfig` 会覆盖现有 `.config`，务必先备份 | `cp .config .config.bak` |
