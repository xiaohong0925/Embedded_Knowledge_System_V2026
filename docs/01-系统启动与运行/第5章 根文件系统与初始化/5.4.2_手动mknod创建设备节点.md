# 5.4.2 手动mknod创建设备节点

> 所属章节：第5章 设备与驱动 > 5.4 节名
> 难度：[B→I] | 预计阅读时间：15分钟

## 本节导读

本节教你使用`mknod`命令手动创建/dev目录下的设备文件，理解主/次设备号的含义，掌握在没有udev的嵌入式环境中手工创建设备节点的完整流程。

---

## 知识点1：mknod命令 [I] ~700字

在Linux系统中，普通文件通过`touch`命令创建，目录通过`mkdir`创建，但设备文件却要用`mknod`（make node）来创建。为什么设备文件这么特殊？因为它们不存储实际数据，而是作为用户空间和内核驱动之间的"通信门牌号"——当用户程序打开`/dev/xxx`时，内核通过设备号找到对应的驱动程序来处理请求。

### mknod基本语法

```bash
mknod [选项] 文件名 类型 主设备号 次设备号
```

**四个参数缺一不可：**

| 参数 | 含义 | 合法取值 |
|------|------|----------|
| 文件名 | 要创建的设备节点路径 | 如`/dev/mydev` |
| 类型 | 设备类型 | `c`字符设备、`b`块设备、`p`FIFO管道 |
| 主设备号 | 标识驱动程序类型 | 0-255（传统）或动态分配 |
| 次设备号 | 标识设备实例 | 通常0-255，依驱动而定 |

**💡 提示**：在现代Linux中，`ls -l`查看设备文件时，会看到类似`crw-rw---- 1 root root 1, 3 Jan 1 1970 /dev/null`的输出，其中`1, 3`就是主设备号和次设备号，用逗号分隔。

### 操作步骤

下面演示创建一个字符设备的完整流程：

1. **确认驱动已加载**：设备节点只是个入口，真正的功能在内核驱动里。先用`cat /proc/devices`查看你的驱动是否注册成功，并记下它的主设备号。

2. **创建节点**：以主设备号250、次设备号0为例

   ```bash
   sudo mknod /dev/mychar c 250 0
   sudo chmod 666 /dev/mychar   # 允许所有用户读写
   ```

3. **验证**：

   ```bash
   ls -l /dev/mychar
   # 应输出：crw-rw-rw- 1 root root 250, 0 ... /dev/mychar
   ```

4. **测试使用**：如果驱动实现了read/write接口，现在就可以用`cat /dev/mychar`或`echo > /dev/mychar`来访问了。

⚠️ **陷阱**：`mknod`创建的节点**不会自动删除**，即使驱动卸载后，/dev下的文件依然存在。这时候访问会得到"No such device"错误（因为驱动不在了，但文件还在）。嵌入式系统启动脚本中，通常在`insmod`加载驱动后紧跟`mknod`，在`rmmod`前用`rm`清理节点。

🔴 **危险**：`mknod`需要root权限。恶意程序可能用`mknod`创建块设备节点来直接读写硬盘（绕过文件系统权限检查），因此某些安全加固的系统会限制`CAP_MKNOD`能力。

## 知识点2：设备号概念 [I] ~600字

### 主次设备号的分工

Linux用**主设备号（Major Number）**和**次设备号（Minor Number）**来精确定位"哪个驱动"处理"哪个具体设备"。这两者的关系，就像"快递公司"和"具体快递柜"：

- **主设备号**（8位/12位传统格式，或32位现代格式）：告诉内核调用哪一个驱动程序。例如所有串口UART驱动的主设备号是4，所有SD卡块驱动是179。
- **次设备号**：告诉驱动程序处理哪一个具体实例。例如`/dev/ttyS0`（次设备号64）和`/dev/ttyS1`（次设备号65）共享同一个串口驱动，但次设备号让驱动知道该操作哪路硬件。

```mermaid

    E -->|主设备号=4| F[串口驱动程序]
```

### 常见设备号速查表

下表列出了嵌入式开发中最常遇到的标准设备号。记住这些，在调试没有udev的最小系统时非常有用：

| 设备文件 | 类型 | 主设备号 | 次设备号 | 功能说明 |
|---------|:----:|:-------:|:-------:|----------|
| /dev/null | c | 1 | 3 | 黑洞，写入丢弃，读取立即EOF |
| /dev/zero | c | 1 | 5 | 零源，无限读取0字节 |
| /dev/random | c | 1 | 8 | 真随机数（阻塞） |
| /dev/urandom | c | 1 | 9 | 伪随机数（非阻塞） |
| /dev/tty | c | 5 | 0 | 当前进程的控制终端 |
| /dev/console | c | 5 | 1 | 系统主控制台 |
| /dev/ptmx | c | 5 | 2 | 伪终端主设备 |
| /dev/loop0 | b | 7 | 0 | 回环块设备（挂载文件用） |
| /dev/sda | b | 8 | 0 | 第一块SCSI/SATA磁盘 |
| /dev/mtd0 | c | 90 | 0 | MTD闪存字符设备 |
| /dev/fb0 | c | 29 | 0 | 帧缓冲区（LCD显示） |
| /dev/input/event0 | c | 13 | 64 | 输入事件（触摸屏/按键） |

💡 **提示**：在目标板上记不住设备号时，可以直接读`/proc/devices`文件。里面分`Character devices:`和`Block devices:`两块，左边是主设备号，右边是驱动名。例如：

```bash
cat /proc/devices
# 输出示例：
# Character devices:
#   1 mem
#   4 /dev/vc/0
#   4 tty
#   5 /dev/tty
# Block devices:
#   1 ramdisk
#   7 loop
#   8 sd
```

⚠️ **陷阱**：主设备号1被多个不相关的驱动共享（`mem`、`null`、`zero`等），这是Linux的**主次设备号复用**机制。它们靠**不同的次设备号**区分功能，而不是靠主设备号一一对应。因此表中`/dev/null`和`/dev/zero`主设备号都是1，但次设备号分别是3和5。

## 知识点3：手动创建示例 [B] ~400字

在嵌入式最小系统（如Buildroot生成的根文件系统、或手工制作的initramfs）中，udev往往不存在，这时启动脚本需要手动创建一组基础设备节点。下面是嵌入式启动阶段最常见的mknod操作集合：

### 操作步骤：创建最小/dev目录

通常在启动脚本（如`/etc/init.d/rcS`）中加入以下内容：

```bash
#!/bin/sh
# 创建必要的设备节点（无udev的最小系统用）

# 创建存储设备目录
cd /dev

# 1. 控制台设备 — 必须有，否则内核启动后没有输出终端
mknod console c 5 1
chmod 622 console

# 2. 空设备 — 脚本重定向常用
mknod null c 1 3
chmod 666 null

# 3. 零源设备 — 擦写、填充数据用
mknod zero c 1 5
chmod 666 zero

# 4. 随机数设备 — 加密、临时文件需要
mknod urandom c 1 9
chmod 666 urandom

# 5. 当前终端 — 很多程序依赖它
mknod tty c 5 0
chmod 666 tty

# 6. 嵌入式特有的MTD闪存分区
mknod mtd0 c 90 0
mknod mtd1 c 90 2
chmod 660 mtd*

# 验证
cd /
ls -l /dev/
```

💡 **提示**：手动创建节点后，建议用`ls -l`逐个检查权限和设备号。嵌入式调试时，如果发现"Permission denied"，九成是权限或所属组不对，而不是驱动问题。

⚠️ **陷阱**：`console`节点的设备号在不同架构下**有差异**！在大多数Linux系统上是`5, 1`，但某些古老的嵌入式内核可能用`4, 0`作为console。如果启动后敲键盘没反应，请检查内核参数中`console=`后面的设备名是否和`mknod`创建的节点匹配。

🔴 **危险**：千万不要手动创建`/`目录或已有运行系统的`/dev`下与现有块设备号冲突的节点！如果误将块设备节点指向正在使用的磁盘分区并直接写入，可能破坏整个文件系统。建议先在`/tmp/testdev`做实验。

## 本节总结

| 概念 | 要点 | 关键操作 |
|------|------|----------|
| mknod命令 | 创建设备文件入口 | `mknod name c/b major minor` |
| 主设备号 | 决定由哪个驱动处理 | 查`/proc/devices`获取 |
| 次设备号 | 决定驱动操作哪个实例 | 由驱动文档或源码确定 |
| 字符设备 | 顺序访问（串口、GPIO） | 类型参数用`c` |
| 块设备 | 随机访问（磁盘、Flash） | 类型参数用`b` |
| 最小系统/dev | 无udev时手动创建 | console、null、zero、urandom必备 |
