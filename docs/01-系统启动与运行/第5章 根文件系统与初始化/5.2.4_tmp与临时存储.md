# 5.2.4 /tmp与临时存储

> 所属章节：第5章 根文件系统 > 5.2 目录结构
> 难度：[B→B] | 预计阅读时间：10分钟

## 本节导读

本节讲解嵌入式Linux中 `/tmp` 目录背后的 **tmpfs** 文件系统原理，以及如何管理临时存储空间。学完本节，你将能够正确挂载内存临时文件系统，并掌握避免内存溢出的实用技巧。

---

## 知识点1：tmpfs——内存里的"临时硬盘" [B] ~500字

### 1.1 什么是tmpfs

在嵌入式设备中，硬盘或Flash存储的擦写次数有限。`/tmp` 目录用来存放临时文件——这些文件重启后不需要保留。Linux提供了一种巧妙的方案：**tmpfs**，一种基于内存（RAM）的虚拟文件系统。

把 tmpfs 理解为"用内存模拟的硬盘分区"：读写速度极快，但断电或重启后数据全部消失。

### 1.2 如何挂载tmpfs到/tmp

大多数Linux发行版已默认将 `/tmp` 挂载为tmpfs。在嵌入式系统中，你可能需要手动配置。

#### 操作步骤

1. **查看当前挂载情况**
2. **手动挂载tmpfs（测试用）**
3. **配置开机自动挂载**

### 代码示例

```bash
# 查看 /tmp 是否已经挂载为 tmpfs
df -h /tmp

# 手动挂载：上限64MB
tmpfs 到 /tmp
mount -t tmpfs -o size=64M tmpfs /tmp

# 验证挂载成功
df -h /tmp
# 预期输出类似：
# Filesystem      Size  Used Avail Use% Mounted on
# tmpfs            64M     0   64M   0% /tmp

# 写入测试文件
echo "hello" > /tmp/test.txt
ls -l /tmp/test.txt

# 查看内存使用情况（tmpfs占用计入内存）
free -h
```

### 关键参数详解

| 挂载参数 | 含义 | 常用值 | 说明 |
|---------|------|--------|------|
| `size` | 容量上限 | `64M`、`50%` | 硬性上限，可超卖 |
| `nr_inodes` | 最大文件数 | `1000` | 限制inode数量 |
| `mode` | 目录权限 | `1777` | 建议sticky位，防误删 |
| `uid` / `gid` | 属主/属组 | `0` | 指定挂载点所有者 |

[表1：tmpfs挂载参数速查表]

### 常见错误

⚠️ **陷阱1：忘记限制size**
如果不设 `size` 参数，tmpfs默认最多可用**物理内存的一半**。嵌入式设备内存紧张，一个失控的程序往 `/tmp` 狂写日志，可能导致系统崩溃。

💡 **提示**：生产环境务必显式设置 `size`，并配合 `df -h /tmp` 监控。

⚠️ **陷阱2：认为tmpfs完全不占磁盘**
tmpfs默认允许**交换到swap分区**。如果启用了swap，tmpfs内容可能"溢"到磁盘，违背性能预期。

🔴 **危险：嵌入式设备通常没有swap分区！**
这意味着tmpfs只能占物理内存。一旦 `/tmp` 写满size上限，再写会报 "No space left on device"——即使磁盘还有空间。

## 知识点2：临时存储管理——用好/tmp [B] ~400字

### 2.1 /tmp 的典型用途

| 使用场景 | 示例 | 注意事项 |
|---------|------|---------|
| 程序临时缓存 | 编译中间文件 `.o` | 编译前确保空间充足 |
| 进程间通信 | Unix域套接字文件 | 重启后自动清理，无需维护 |
| 日志缓冲 | 应用调试日志 | 控制日志大小，防止撑爆内存 |
| 锁文件 | `/tmp/myapp.lock` | 崩溃后锁残留，需设计超时机制 |

[表2：/tmp常见使用场景]

### 2.2 自动清理策略

临时文件如果不清理，内存会持续占用。常用策略有三种：

1. **系统级清理**：systemd定时服务 `systemd-tmpfiles`，或传统cron任务
2. **应用自清理**：程序退出前删除自己的临时文件
3. **重启清零**：tmpfs特性，重启后自然清空（最简单粗暴）

#### 配置systemd-tmpfiles示例

```bash
# 创建清理规则配置文件
# /etc/tmpfiles.d/myapp.conf

# 类型  路径          权限  属主  属组  年龄
d       /tmp/myapp    0755  root  root  - -
D       /tmp/myapp/*  0644  root  root  1h -
```

💡 **提示**：`D` 表示目录内容按年龄自动清理，`1h` 表示1小时前的文件将被删除。

### 2.3 内存占用监控

```bash
# 查看tmpfs实际占用
du -sh /tmp

# 查看整体内存使用（包含tmpfs）
free -h

# 查看各tmpfs挂载点
cat /proc/mounts | grep tmpfs
```

⚠️ **陷阱：free命令中的"shared"**
tmpfs占用的内存在 `free` 输出中会计入 `used`，但不会单独标注。如果内存莫名占用高，检查 `/tmp`、`/run`、`/dev/shm` 等tmpfs挂载点。

## 本节总结

| 概念 | 要点 | 操作 |
|------|------|------|
| tmpfs本质 | 基于RAM的虚拟文件系统 | `mount -t tmpfs` |
| 容量限制 | 必须设置`size`防止耗尽内存 | `-o size=64M` |
| 数据持久性 | 重启/断电即丢失 | 勿存重要数据 |
| 自动清理 | systemd-tmpfiles或应用自管理 | 配置文件规则 |
| 内存监控 | tmpfs占用计入物理内存 | `df -h` + `du -sh` |
