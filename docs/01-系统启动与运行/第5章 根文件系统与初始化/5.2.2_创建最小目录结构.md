# 5.2.2 创建最小目录结构

> 所属章节：第5章 根文件系统构建 > 5.2 最小根文件系统
> 难度：[B] | 预计阅读时间：12分钟

## 本节导读
本节教你用一条 `mkdir -p` 命令搭建出嵌入式Linux能启动的最小目录骨架，并学会用 `tree` 验证目录结构是否完整。学完后，你将拥有一个"空但完整"的根文件系统雏形。

---

## 知识点1：创建目录结构 [B] ~500字

Linux内核启动成功后，第一个用户态进程（通常是 `/sbin/init` 或自定义的 `/init`）会从根文件系统的根目录 `/` 开始运行。为了让内核找到它、让各类程序找到配置和设备，我们必须提前准备好一套**最小目录结构**。

### 创建命令

在构建目录（假设为 `~/rootfs`）下执行一条命令即可：

```bash
cd ~/rootfs
mkdir -p bin sbin etc dev proc sys tmp usr/bin usr/sbin var
```

`mkdir -p` 的 `-p` 选项非常关键：它会**自动创建父目录**。例如 `usr/bin` 会先创建 `usr/`，再在其下创建 `bin/`。没有这个选项，嵌套路径会报错。

### 各目录用途速查

| 目录 | 全称/含义 | 存放内容 | 是否必须 |
|------|-----------|----------|----------|
| `bin/` | binaries | 普通用户命令（如 `sh`、`ls`、`mount`） | ✅ 是 |
| `sbin/` | system binaries | 系统管理命令（如 `init`、`reboot`） | ✅ 是 |
| `etc/` | etcetera | 配置文件（如 `inittab`、`fstab`） | ✅ 是 |
| `dev/` | devices | 设备节点（如 `null`、`console`） | ✅ 是 |
| `proc/` | process | proc虚拟文件系统挂载点 | ✅ 是 |
| `sys/` | system | sysfs虚拟文件系统挂载点 | ✅ 是 |
| `tmp/` | temporary | 临时文件 | ✅ 是 |
| `usr/bin/` | user binaries | 非关键的普通用户命令 | 建议 |
| `usr/sbin/` | user system binaries | 非关键的系统命令 | 建议 |
| `var/` | variable | 日志、运行时数据 | 建议 |

💡 **提示**：上表中的 "是否必须" 是从**最小可启动**角度判定的。`usr` 和 `var` 下即使没有文件，目录本身也应存在，因为很多脚本和程序默认向这些路径写数据，缺少会导致运行时错误。

### /init 的重要性

内核挂载根文件系统后，会按固定顺序寻找第一个用户态程序：

1. 优先寻找 `/init`
2. 找不到则尝试 `/sbin/init`
3. 都找不到 → **启动失败**，报 `kernel panic`

因此，哪怕 `/init` 只是一个指向 `/sbin/init` 的符号链接，它的存在也至关重要。我们后续小节会动手创建它。

⚠️ **陷阱**：不要把 `init` 放在 `/bin/` 或随便某个目录下，内核只认固定的几个路径。位置错了，哪怕文件内容正确也启动不了。

## 知识点2：目录树验证 [B] ~400字

目录创建完成后，不能凭"感觉"确认。下面介绍两种验证方法。

### 方法1：使用 tree 命令（推荐）

`tree` 命令以树状图直观展示目录结构，是验证的最佳工具：

```bash
cd ~/rootfs
tree -L 2 -d
```

`-L 2` 限制只显示2层深度，`-d` 只显示目录（不显示文件）。期望输出如下：

```
.
├── bin
├── dev
├── etc
├── proc
├── sbin
├── sys
├── tmp
├── usr
│   ├── bin
│   └── sbin
└── var

10 directories, 0 files
```

如果输出显示的目录数量与上表一致，说明骨架搭建成功。

### 方法2：使用 ls + 手动检查

若系统没有安装 `tree`，可用 `ls` 替代：

```bash
cd ~/rootfs
ls -F | grep '/'
```

这条命令只列出以 `/` 结尾的条目（即目录），然后逐个确认 `usr/bin`、`usr/sbin` 是否存在。

### 自动化检查脚本

为了日后构建时一键验证，可以写一个检查清单脚本：

```bash
#!/bin/bash
DIRS="bin sbin etc dev proc sys tmp usr/bin usr/sbin var"
for d in $DIRS; do
    if [ -d "$d" ]; then
        echo "✓ $d"
    else
        echo "✗ $d 缺失!"
    fi
done
```

保存为 `check_dirs.sh`，在 `~/rootfs` 目录下运行，所有项目都显示 ✓ 即为通过。

⚠️ **陷阱**：`tree` 命令在某些精简系统上默认未安装。如果提示 `command not found`，可以用包管理器安装：`sudo apt install tree`（Debian/Ubuntu）或 `sudo yum install tree`（RHEL/CentOS）。

💡 **提示**：建议将 `check_dirs.sh` 纳入你的项目版本控制，每次重新构建根文件系统后都运行一遍，确保不会因为手误漏建某个目录。

## 本节总结

| 操作 | 命令/要点 | 验证方法 |
|------|-----------|----------|
| 创建骨架 | `mkdir -p bin sbin etc dev proc sys tmp usr/bin usr/sbin var` | `tree -L 2 -d` |
| 嵌套目录 | 依赖 `-p` 自动创建父目录 | 检查 `usr/bin`、`usr/sbin` |
| init路径 | 必须是 `/init` 或 `/sbin/init` | 后续小节放置 init 程序 |
| 重复验证 | 编写 `check_dirs.sh` 脚本 | 运行脚本确认 10 个目录齐全 |

通过本节，我们已经搭建好了根文件系统的"空房子"。接下来两节将依次往这些目录里填充**设备节点**（5.2.3）和**初始化程序**（5.2.4），让房子真正"住"起来。
