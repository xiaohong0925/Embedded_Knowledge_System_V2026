# I2C历史演进与未来展望

<span class="badge-i">[Intermediate]</span>

<span class="red">I2C（Inter-Integrated Circuit）</span> 由 Philips 于 1982 年发明，是最广泛使用的低速串行总线。

---

## <strong>历史演进</strong>

- <span class="green">1982 年 I2C 1.0</span> — 100kbps，Philips 内部总线<br>
- <span class="green">1992 年 I2C 2.0</span> — 引入 Fast Mode 400kbps<br>
- <span class="green">2000 年 I2C 3.0</span> — High Speed Mode 3.4Mbps<br>
- <span class="green">2000s SMBus</span> — Intel 简化子集，电源管理<br>
- <span class="green">2000s PMBus</span> — 电源管理总线扩展<br>
- <span class="green">2016 年 I3C</span> — MIPI 继任者，12.5Mbps，向后兼容

## <strong>从I2C到I3C的过渡</strong>

| 特性 | I2C | I3C |
|------|-----|-----|
| 速率 | 3.4Mbps | 12.5Mbps+ |
| 功耗 | 开漏，静态电流 | 推挽，动态驱动 |
| 中断 | 需要额外线 | 带内中断（IBI） |
| 热管理 | 无 | 内置温度报告 |

<span class="blue">I3C 目标不是完全替代 I2C</span>，而是成为新一代移动/IoT设备的首选。
