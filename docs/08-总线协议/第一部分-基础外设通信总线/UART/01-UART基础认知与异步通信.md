## UART 的诞生：异步通信的鼻祖

---

### <strong>为什么需要 UART：无需时钟线的自由通信</strong>

<span class="red">UART</span>的历史可追溯到<span class="green">1960年代</span>的电传打字机（Teletype），
<br>
是<span class="blue">"最早的标准化串行通信协议"</span>。
<br>

与 I2C/SPI 的关键差异：
<br>
* I2C/SPI 需要共享时钟线（SCL/SCK），UART<span class="blue">不需要时钟线</span>
<br>
* 收发双方通过<span class="blue">"约定波特率"</span>同步，而不是物理时钟
<br>
* 距离更远（RS-232 可达 15m，RS-485 可达 1200m）
<br>

<span class="blue">类比：I2C/SPI 如同"两人一起跳舞"——必须踩着同一个节拍；UART 如同"各自哼歌"——事先约定好速度，然后各自独立发送。</span>
<br>

---

### <strong>UART 帧格式：起始位 + 数据位 + 校验位 + 停止位</strong>

<span class="red">UART 帧</span>的结构如同"信封"：
<br>

```text
空闲:  ─────────────────────────────────────────
起始:  ─┐                                      
数据:   │  D0   D1   D2   D3   D4   D5   D6   D7
校验:   │                                      P
停止:   │                                      └──┐
       └─────────────────────────────────────────┘

1 起始位(0) + 8 数据位(LSB first) + 1 校验位 + 1 停止位(1)
```

| 字段 | 长度 | 说明 |
| --- | --- | --- |
| 起始位 | 1 bit | 低电平，标志帧开始 |
| 数据位 | 5~9 bit | 通常 8 bit，LSB 先传 |
| 校验位 | 0~1 bit | 奇校验/偶校验/无校验 |
| 停止位 | 1~2 bit | 高电平，标志帧结束 |

<span class="blue">10 位传输 1 字节：1 起始 + 8 数据 + 1 停止 = 10 bit/byte。9600 波特率实际吞吐量为 9600/10 = 960 byte/s。</span>
<br>

---

### <strong>波特率与误差容忍</strong>

<span class="red">波特率</span>是 UART 的核心参数：
<br>

| 波特率 | 位时间 | 典型应用 |
| --- | --- | --- |
| 9600 | 104.2 μs | 调试串口、慢速模组 |
| 115200 | 8.68 μs | 高速调试、蓝牙透传 |
| 921600 | 1.09 μs | 高速下载、WiFi 模组 |

<span class="blue">收发双方波特率误差必须 < 2%，否则采样点偏移导致误码。这就是为什么 UART 需要精确晶振或波特率发生器。</span>
<br>

```c
// STM32 UART 波特率配置（寄存器级）
// USARTDIV = fCK / (16 × BaudRate)
// fCK = 84MHz, BaudRate = 115200
// USARTDIV = 84000000 / (16 * 115200) = 45.57
USART1->BRR = 45;  // 整数部分
```

## 本章小结

| 概念 | 一句话总结 |
| --- | --- |
| UART | 异步串口，无需时钟线，约定波特率通信 |
| 帧格式 | 起始位(0) + 数据位 + 校验位 + 停止位(1) |
| 波特率 | 每秒传输的 bit 数，误差需 < 2% |
| RS-232 | 单端信号，15m 距离 |
| RS-485 | 差分信号，1200m 距离，支持多主 |
