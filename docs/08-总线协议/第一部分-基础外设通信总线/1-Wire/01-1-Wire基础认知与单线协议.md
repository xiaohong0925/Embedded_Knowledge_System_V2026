# 1-Wire 基础认知与单线协议

<span class="red">1-Wire 是 Maxim Integrated（现 Analog Devices）推出的单线串行通信协议，仅用一根数据线（DQ）同时完成供电和数据传输，在温度传感、身份识别等领域广泛应用。</span>

类比：摩斯电码通过"点"和"划"两种不同时长的脉冲传递信息，1-Wire 通过不同时隙（time slot）区分逻辑 0 和逻辑 1，一根线足以完成全部通信。

---

## 为什么单线足够

<span class="red">1-Wire 的核心设计思想是用时间（时隙）代替额外的信号线区分信息。每个时隙固定 60μs，槽内电平高低表示 0 或 1。</span>

### 时隙 vs 多线

传统总线用多根线区分功能：I2C 用 SDA+SCL，SPI 用四根线。1-Wire 将"时钟信息"编码在时隙宽度中，接收方在约定时刻采样，无需独立时钟线。

| 总线 | 信号线 | 时钟方式 | 最远距离 |
|------|--------|----------|----------|
| I2C | 2 | SCL 同步 | ≤1m |
| SPI | 4 | SCK 同步 | ≤30cm |
| 1-Wire | 1 | 时隙异步 | ≤100m |

单线设计的代价是速率极低（标准模式 16.3kbps），远低于 I2C/SPI。但温度传感器、门禁卡等场景数据量极小，速率不是瓶颈。

### 拓扑结构

1-Wire 支持线性拓扑和星型拓扑。线性拓扑中设备串联在一条总线上，星型拓扑中从主干分支出多个支线。星型拓扑的支线长度不宜超过 3 米，否则信号反射会干扰通信。

---

## 单线架构：数据+电源

<span class="red">1-Wire 的数据线 DQ 同时承担供电功能，设备可选择寄生电源模式（从数据线偷电）或外部供电模式（独立 VCC）。</span>

### 寄生电源模式

主机通过 4.7kΩ 上拉电阻将 DQ 维持在高电平。设备内部集成储能电容（约 800pF），在 DQ 为高时充电，在 DQ 为低时用存储的能量维持工作。

| 模式 | 接线 | 适用场景 |
|------|------|----------|
| 寄生电源 | 仅 DQ+GND | 布线极简、低功耗 |
| 外部供电 | DQ+VCC+GND | 大电流操作（如温度转换） |

<span class="blue">易错点：寄生电源模式下进行温度转换（A/D 采样耗电大）时，必须确保主机强上拉（DQ 直接拉高，绕过 4.7kΩ）持续供电，否则设备会因欠压复位。</span>

### 强上拉电路

强上拉通常用 MOSFET 短路 4.7kΩ 电阻，将 DQ 直接拉到 VCC。转换期间（DS18B20 的 750ms）必须保持强上拉，转换完成后恢复正常上拉。电路实现上可用 GPIO 控制 MOSFET 栅极。部分教程推荐用 GPIO 直接推挽输出强上拉，但 GPIO 驱动电流有限，MOSFET 方案更可靠。

## ROM-ID：64 位唯一标识

<span class="red">每个 1-Wire 设备出厂时烧录 64 位唯一 ROM-ID，前 8 位为家族码（Family Code），后 48 位为序列号，最后 8 位为 CRC 校验。</span>

### 64 位结构

```
[7:0]   Family Code    (8 bit)  设备类型标识
[55:8]  Serial Number  (48 bit) 唯一序列号
[63:56] CRC8           (8 bit)  前 56 位 CRC 校验
```

| Family Code | 设备类型 |
|-------------|----------|
| 0x10 | DS18S20 温度传感器 |
| 0x28 | DS18B20 温度传感器 |
| 0x01 | DS1990A 序列号 iButton |
| 0x2D | DS2431 1Kb EEPROM |
| 0x3B | DS1825 温度/报警传感器 |
| 0x42 | DS28EA00 带链路功能的温度传感器 |

主机通过 Match ROM（0x55）命令加 64 位 ROM-ID 选中特定设备，或通过 Skip ROM（0xCC）广播命令。

### CRC8 校验

ROM-ID 的最后 8 位是 CRC8 校验值，多项式为 `x^8 + x^5 + x^4 + 1`（0x31）。主机读取 ROM-ID 后应验证 CRC，若校验失败说明通信受干扰，需重新读取。DS18B20 的 Scratchpad 数据最后也有 CRC8，用于验证温度读取完整性。

## 速度模式：Standard / Overdrive

<span class="red">1-Wire 定义两种速度模式：Standard（16.3kbps）兼容所有设备，Overdrive（142kbps）仅支持部分高速设备。</span>

### 模式对比

| 参数 | Standard | Overdrive |
|------|----------|-----------|
| 时隙宽度 | 60-120μs | 6-16μs |
| 复位低电平 | 480μs | 48μs |
| 存在脉冲 | 60-240μs | 8-24μs |
| 速率 | 16.3 kbps | 142 kbps |
| 兼容性 | 全部设备 | 仅高速设备 |

进入 Overdrive 模式需先发送 Standard 速率的 Overdrive Skip ROM（0x3C）命令，此后所有通信切换到高速时隙。

## DS18B20 温度传感器

<span class="red">DS18B20 是最常用的 1-Wire 设备，支持 -55°C ~ +125°C 测温，分辨率可配置为 9-12 位，转换时间从 93.75ms 到 750ms。</span>

### 分辨率配置

| 分辨率 | 分辨率值 | 转换时间 | 精度 |
|--------|----------|----------|------|
| 9 bit  | 0.5°C | 93.75 ms | ±0.5°C |
| 10 bit | 0.25°C | 187.5 ms | ±0.25°C |
| 11 bit | 0.125°C | 375 ms | ±0.125°C |
| 12 bit | 0.0625°C | 750 ms | ±0.0625°C |

分辨率通过 Scratchpad 配置寄存器的 R0/R1 位设置。12 位模式下温度数据为 16 位有符号数，低 4 位为小数部分。

### 温度读取流程

1. 发送复位脉冲，检测存在脉冲
2. 发送 Skip ROM（0xCC）或 Match ROM（0x55）
3. 发送 Convert T（0x44）启动温度转换
4. 等待转换完成（12 位模式 750ms）
5. 再次复位，发送 Read Scratchpad（0xBE）
6. 读取 9 字节 Scratchpad 数据
7. 验证第 9 字节 CRC8

Scratchpad 9 字节布局：温度 LSB、温度 MSB、TH 报警上限、TL 报警下限、配置寄存器、保留×3、CRC8。

<span class="blue">易错点：DS18B20 上电默认 12 位分辨率，若应用不需要 0.0625°C 精度，应降低分辨率以缩短转换时间和功耗。</span>

---

## 小节

- 1-Wire 用一根线完成通信和供电，代价是速率极低（16.3kbps）。
- 寄生电源模式省掉 VCC 线，但大电流操作需要主机强上拉支持。
- 64 位 ROM-ID 是全球唯一的设备标识，家族码区分设备类型。
- DS18B20 是最典型的 1-Wire 应用，分辨率与转换时间成正比。
- 强上拉电路是寄生电源模式可靠工作的关键，不可忽视。
