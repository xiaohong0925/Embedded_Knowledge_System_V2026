## 01-AMBA 是什么与为什么存在 **[B]

> <span class="badge-b">B</span>

### <strong>片上总线的核心定义</strong>

<span class="badge-b">B</span>

- 什么是片上总线（On-Chip Bus）
  - SoC 内部 IP 核之间的"高速公路系统"：CPU、GPU、DMA、外设控制器通过总线互连
  - 与片外总线（PCIe、USB）的区别：距离短、延迟低、功耗敏感、无需 PHY 层
  - 没有总线 = IP 核之间点对点直连，面积爆炸、布线灾难
- 为什么需要标准化总线
  - IP 复用：ARM 卖 CPU 核、Synopsys 卖 DMA 核，它们必须讲同一种"语言"才能对接
  - 设计分工：前端团队选 IP，后端团队搭总线，接口一致才能并行工作
  - <span class="blue">总线是 SoC 的骨架，IP 是器官，没有骨架器官就是一团肉。</span>

### <strong>AMBA 的诞生与 ARM 生态绑定</strong>

<span class="badge-b">B</span>

- AMBA = Advanced Microcontroller Bus Architecture，ARM 1996 年推出
- 不是独立标准组织，而是 ARM 为自家 IP 核定义的互连接口
- 生态惯性：用 ARM Cortex-A 就必须会 AMBA，否则买不到/用不了主流 IP
- 开放性：ARM 向第三方 IP 厂商开放许可，Synopsys、Cadence、自研核都可以接 AMBA

### <strong>AMBA 家族成员速览</strong>

<span class="badge-b">B</span>

| 协议 | 定位 | 典型场景 | 速度量级 |
|------|------|----------|----------|
| AXI4/AXI5 | 高性能、多通道 | CPU ↔ DDR、GPU、DMA | GHz 级 |
| AHB | 中等性能、单通道 | CPU ↔ SRAM、Flash 控制器 | ~200MHz |
| APB | 低速、极简 | UART、I2C、GPIO、定时器寄存器 | ~100MHz |
| ACE | 缓存一致性扩展 | 多核 CPU 共享缓存 | 同 AXI |
| CHI | 面向服务器/高性能计算 | 多插芯、多节点 | 更高 |

- 总线层级关系：AXI（主通路）→ AHB（中等外设）→ APB（低速寄存器），通过桥接器互联

## 02-AMBA 演进路线与技术选择逻辑 **[I]

> <span class="badge-i">I</span>

### <strong>从 AHB2 到 AXI4/AXI5 的演进脉络</strong>

<span class="badge-i">I</span>

- AHB2（1999）：单通道流水线，简单高效，适合早期 ARM9 时代
- AXI3（2003）：引入 5 个独立通道（读地址/读数据/写地址/写数据/写响应），支持乱序完成
- AXI4（2010）：简化 AXI3，去除锁定传输，增加 QoS 信号，成为主流
- AXI5（2021）：新增 Trace、PMU、ATB 扩展，面向汽车/安全领域
- 选择逻辑：新设计直接用 AXI4，汽车/功能安全场景考虑 AXI5， legacy 系统维护 AHB2

### <strong>APB 为什么一直不变</strong>

<span class="badge-i">I</span>

- APB2（1999）→ APB3（增加 PREADY/PERROR）→ APB4（增加 PPROT/PSTRB）
- 核心思想：低速外设不需要流水线，单周期握手足够，越简单越省电
- 变化极小：APB 接口 20 年几乎没变，证明"简单够用就不要动"的设计哲学
- <span class="blue">APB 是 AMBA 家族里的老爷爷，什么都不做但就是活着。</span>

### <strong>ACE 与 CHI：缓存一致性扩展</strong>

<span class="badge-i">I</span>

- ACE（AXI Coherency Extensions）：在 AXI4 上加缓存一致性信号，多核共享内存
- CHI（Coherent Hub Interface）：ARM 新架构，面向服务器级多芯互联，取代 ACE
- 嵌入式场景选择：双核 Cortex-A 用 ACE，四核以上或服务器用 CHI

### <strong>AMBA 总线选择决策树</strong>

<span class="badge-i">I</span>

- 第一步：看带宽需求
  - >1GB/s → AXI4/5；100MB/s-1GB/s → AHB；<100MB/s 且是寄存器访问 → APB
- 第二步：看是否需要缓存一致性
  - 多核共享缓存 → ACE/CHI；单核或 DMA 直通 → AXI4 足够
- 第三步：看功耗/面积约束
  - 超低功耗 MCU → AHB + APB 足够，不需要 AXI；复杂多媒体 SoC → AXI + AHB + APB 三级总线

## 03-AMBA 与其他片内总线对比 **[I]

> <span class="badge-i">I</span>
> ⚠️ 不做"优劣"对比，只做"适用场景"对照

### <strong>AMBA vs TileLink vs Wishbone 定位差异</strong>

<span class="badge-i">I</span>

| 维度 | AMBA | TileLink | Wishbone |
|------|------|----------|----------|
| 出身 | ARM 商业生态 | RISC-V 开源生态 | OpenCores 开源 |
| 设计目标 | 高性能 SoC 商业化 | 处理器/加速器一致性 | FPGA 简单互连 |
| 协议复杂度 | 高（数百页 spec） | 中（缓存一致性为核心） | 低（几十页 spec） |
| 缓存一致性 | ACE/CHI 原生支持 | 原生支持（TL-C） | 不支持 |
| 典型用户 | ARM SoC 厂商 | RISC-V 芯片公司 | 学术界/FPGA 项目 |

### <strong>什么时候不选 AMBA</strong>

<span class="badge-i">I</span>

- 自研 RISC-V 处理器 + 自定义加速器 → TileLink 更自然（SiFive 全家桶）
- 教学/FPGA 原型验证 → Wishbone 简单到能手写 RTL
- 非 ARM 生态的旧架构维护（MIPS、PowerPC 自有总线）

### <strong>最低门槛认知实验</strong>

<span class="badge-i">I</span>

- 打开一份 ARM CoreLink NIC-400 数据手册，数一下它支持多少个 AXI/ AHB/ APB 端口
- 理解：总线矩阵不是一对一连接，而是多主多从的交叉开关
- 观察：APB 端口数量总是最多，因为低速外设最多
