## 01-AMBA 是什么与为什么存在 **[B→I]**

> <span class="badge-b">B</span> → <span class="badge-i">I</span>
> 内容类型：场景描述 + 原理解析
> 预计密度：中

### <strong>片上总线的核心定义</strong>

<span class="badge-b">B</span>

片上总线（On-Chip Bus）是 SoC（System on Chip，系统级芯片）内部各个 IP 核之间的互联通道。可以把 SoC 想象成一座微型城市：CPU 是市政府，GPU 是发电厂，DDR 控制器是仓库，UART/SPI 是邮局。这些"建筑"之间需要道路连接，片上总线就是这些道路。

如果没有标准化的总线，每个 IP 核之间需要点对点拉线。假设 SoC 里有 10 个 IP 核，两两互联需要 C(10,2)=45 组信号线，布线面积爆炸、时序难以收敛。总线的作用就是提供一个"高速公路系统"，所有 IP 核都按统一接口接进来，由总线矩阵负责路由。

片上总线与片外总线（如 PCIe、USB、以太网）有本质区别：
- **距离**：片内距离微米级，片外厘米到米级
- **PHY 层**：片内没有物理收发器，直接是数字逻辑连线；片外需要 SerDes、差分对等复杂 PHY
- **功耗**：片内总线可以精细时钟门控，功耗敏感；片外总线功耗主要来自驱动器和线缆
- **延迟**：片内总线延迟纳秒级，片外总线延迟微秒到毫秒级

<span class="red">没有总线 = IP 核之间点对点直连，面积爆炸、布线灾难。</span>

### <strong>为什么需要标准化总线</strong>

<span class="badge-i">I</span>

SoC 设计是一个高度分工的产业。ARM 设计 CPU 核，Synopsys 设计 DDR 控制器，Imagination 设计 GPU，Cadence 设计 DMA 核。这些来自不同公司的 IP 核必须能"说同一种语言"才能互联。

标准化总线的价值体现在三个层面：

**1. IP 复用**
验证一个 IP 核需要数月时间和数百万美元。如果每次换项目都重新设计接口，成本不可接受。有了 AMBA 标准，ARM 的 Cortex-A53 核可以直接卖给苹果、高通、联发科、海思，不需要为每一家定制接口。

**2. 设计分工并行化**
前端团队负责选 IP 和搭总线拓扑，后端团队负责布线布局。如果接口不统一，前端和后端必须串行工作：前端先定好一个 IP 的接口，后端才能开始布线，效率极低。

**3. 工具链生态**
EDA 工具（如 Cadence SoC Encounter、Synopsys Design Compiler）对标准总线有优化策略。非标准接口无法享受自动时序优化、功耗分析等功能。

<span class="blue">总线是 SoC 的骨架，IP 是器官，没有骨架器官就是一团肉。</span>

### <strong>典型应用场景速览</strong>

<span class="badge-b">B</span>

| 芯片 | 总线架构 | 说明 |
|------|----------|------|
| 手机 SoC（骁龙/天玑） | AXI4 + AHB + APB 三级 | CPU/GPU/NPU 走 AXI，外设走 AHB/APB |
| 车载 MCU（i.MX8） | AXI4 + ACE | 多核 Cortex-A + Cortex-M，需要缓存一致性 |
| MCU（STM32） | AHB + APB | 单片机无需 AXI 的高带宽，AHB 足够 |
| RISC-V 芯片（SiFive） | TileLink | 非 ARM 生态，使用开源总线 |

### <strong>从 AHB2 到 AXI4/AXI5 的演进脉络</strong>

<span class="badge-i">I</span>

AMBA 协议从 1996 年诞生至今经历了多代演进，每一代都对应着 SoC 性能需求的跃升。

**AMBA2（1999）**
包含 ASB（已淘汰）、AHB 和 APB。AHB2 采用单通道共享总线，支持流水线突发传输，最高频率约 200MHz。这一代对应 ARM9 时代，手机还是功能机，SoC 复杂度低，AHB 足够。

**AMBA3（2003）**
引入 AXI3，这是革命性的升级。AXI3 把地址和数据彻底分离为 5 个独立通道（读地址、读数据、写地址、写数据、写响应），支持读写并发、乱序完成、可变突发长度。这一代对应 ARM11 / 早期 Cortex-A 时代，智能手机开始出现多媒体需求。

**AMBA4（2010）**
AXI4 简化 AXI3，去除不常用的锁定传输（locked transfer），增加 QoS（Quality of Service）信号，定义 AXI4-Lite（简化版，无突发）和 AXI4-Stream（无地址，纯数据流）。AXI4 成为当前绝对主流，对应 Cortex-A15/A53/A72 时代。

**AMBA5（2021）**
AXI5 不是 AXI4 的替代，而是扩展。新增 Trace 信号（ATB 集成）、PMU 信号（性能监控）、安全扩展（TrustZone 细化）。同时推出 CHI（Coherent Hub Interface）替代 ACE，面向服务器级多芯互联。

选择建议：
- <span class="green">新项目直接用 AXI4</span>，生态最全、IP 最多
- <span class="green">汽车/功能安全场景考虑 AXI5</span>，有安全扩展和冗余建议
- <span class="green">Legacy 系统维护用 AHB2</span>，如旧款 MCU 迭代

### <strong>APB 为什么一直不变</strong>

<span class="badge-i">I</span>

APB 从 1999 年的 APB2 到 2010 年的 APB4，核心时序几乎没有变化，只增加了几个可选信号。

这是因为 APB 的设计目标本身就是"极简寄存器访问"：
- UART 波特率寄存器：32bit，写一次管很久
- GPIO 方向寄存器：配置一次，运行中很少改
- 定时器初值寄存器：设置后自动递减

这些操作都是低频、小数据量，不需要突发、不需要乱序、不需要高频率。APB 的 2 周期固定时序（SETUP + ENABLE）足够，而且实现面积只有 AXI 的 1/5。

变化历史：
- APB2：基础版，无等待、无错误报告
- APB3：加 PREADY（允许 Slave 慢响应）和 PSLVERR（错误报告）
- APB4：加 PPROT（TrustZone/特权级标记）和 PSTRB（字节选通，支持窄传输）

<span class="blue">APB 是 AMBA 家族里的老爷爷，什么都不做但就是活着。</span>

### <strong>ACE 与 CHI：缓存一致性扩展</strong>

<span class="badge-i">I</span>

多核 SoC 中，每个 CPU 核有私有 L1 cache（有时还有 L2 cache）。如果核 A 修改了 cache 里的变量，核 B 读内存可能拿到旧值——这就是缓存不一致。

ACE（AXI Coherency Extensions）在 AXI4 基础上增加信号：
- 缓存状态传递：Shareable/Inner/Outer 域
- 窥探（Snoop）：一个核修改 cache line，通知其他核失效
- 脏写回（Dirty Writeback）：修改过的数据写回内存

CHI 是 ARM 的新一代一致性协议，与 ACE 的关系类似 AXI 与 AHB——CHI 面向更高性能场景。但在嵌入式领域，双核/四核 Cortex-A 用 ACE 足够，八核以上才需要 CHI。

### <strong>AMBA vs TileLink vs Wishbone 定位差异</strong>

<span class="badge-i">I</span>

| 维度 | AMBA | TileLink | Wishbone |
|------|------|----------|----------|
| 出身 | ARM 商业生态 | RISC-V/SiFive 开源生态 | OpenCores 开源 |
| 设计目标 | 高性能 SoC 商业化 | 处理器/加速器一致性互联 | FPGA/教学简单互连 |
| 协议页数 | ~300 页 | ~100 页 | ~30 页 |
| 缓存一致性 | ACE/CHI 扩展 | 原生 TL-C | 不支持 |
| 开源实现 | CoreLink 闭源，spec 公开 | Rocket Chip/Chipyard 全开源 | OpenCores 全开源 |
| 典型频率 | ASIC GHz 级 | ASIC/FPGA 都支持 | FPGA 50-100MHz |

<span class="red">不做优劣对比，只说明：ARM 生态选 AMBA，RISC-V 生态选 TileLink，教学/FPGA 选 Wishbone。</span>

### <strong>什么时候不选 AMBA</strong>

<span class="badge-i">I</span>

- 自研 RISC-V 处理器 + 自定义加速器 → TileLink 更自然，SiFive 全家桶原生支持
- 教学/FPGA 原型验证 → Wishbone 简单到能手写 RTL，OpenCores 有成百个现成 IP
- 非 ARM 生态的旧架构维护（MIPS、PowerPC）→ 各自有原生总线

### <strong>最低门槛认知实验</strong>

<span class="badge-i">I</span>

打开一份 ARM CoreLink NIC-400 数据手册，翻到框图页：
- 数一下有多少个 AXI 主端口（通常 4-8 个）
- 数一下有多少个 AHB 从端口（通常 2-4 个）
- 数一下有多少个 APB 从端口（通常 20-40 个）

观察规律：APB 端口数量总是最多，因为低速外设最多；AXI 端口最少但带宽最高。这就是"三级总线"的现实依据。
