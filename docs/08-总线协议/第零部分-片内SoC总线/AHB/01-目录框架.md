## 01-AHB基础认知与架构.md **[B→I]**

> <span class="badge-b">B</span> → <span class="badge-i">I</span>

### <strong>AHB 核心定义与价值</strong>

<span class="badge-b">B</span>

- 什么是 AHB（Advanced High-performance Bus）
  - AMBA2 时代定义的高性能总线，1999 年推出
  - 单通道共享总线：地址和数据共用一组信号线，分时复用
  - 支持突发传输、多主仲裁、流水线操作
- 为什么还需要 AHB（AXI 不是更好吗？）
  - 面积/功耗敏感：AHB 接口比 AXI 小 30-40%，低功耗 MCU 用 AHB 更省面积
  - 复杂度适中：不需要乱序完成的场景，AHB 足够且更容易验证
  - legacy 生态：大量 ARM9、Cortex-M 系列 SoC 仍使用 AHB
  - <span class="blue">AHB 是 AXI 的"经济适用房"，不是豪华别墅，但住得舒服。</span>
- 典型芯片中的 AHB
  - STM32F4/F7/H7：Cortex-M4/M7 内核通过 AHB 总线矩阵连接 SRAM、Flash、DMA、APB 桥
  - i.MX6ULL：Cortex-A7 的 L2 cache 控制器后接 AHB 总线矩阵到各类外设

### <strong>AHB 与 AXI 的本质差异</strong>

<span class="badge-b">B</span>

| 特性 | AHB | AXI4 |
|------|-----|------|
| 通道数 | 1（地址+数据复用） | 5（读写分离） |
| 读写并发 | 半双工，分时 | 全双工，并发 |
| 突发长度 | 4/8/16 beat（固定） | 1-256 beat（灵活） |
| 乱序完成 | 不支持 | 支持（ID 路由） |
| 等待周期 | HREADY 插入等待 | 每个通道独立握手 |
| 面积 | 小 | 大 |

- 选择逻辑：带宽需求 < 200MB/s + 无需乱序 + 面积敏感 → AHB；否则 → AXI

### <strong>AHB 总线信号速览</strong>

<span class="badge-i">I</span>

- HCLK / HRESETn：时钟与复位
- HADDR[31:0]：32 位地址总线
- HWDATA / HRDATA：写数据 / 读数据
- HWRITE：读写方向（1=写，0=读）
- HSIZE[2:0]：传输大小（Byte/Halfword/Word/...）
- HBURST[2:0]：突发类型（SINGLE/INCR/WRAP4/INCR4/...）
- HTRANS[1:0]：传输类型（IDLE/BUSY/NONSEQ/SEQ）
- HREADY：Slave 就绪信号（插入等待周期）
- HRESP[1:0]：响应状态（OKAY/ERROR/RETRY/SPLIT）

## 02-AHB传输时序与流水线.md **[I]**

> <span class="badge-i">I</span>

### <strong>基本传输时序</strong>

<span class="badge-i">I</span>

- 无等待传输：2 周期完成一个 burst beat
  - T1：Master 放地址 + 控制信号，Slave 采样地址
  - T2：Slave 放读数据 / Master 放写数据，同时 HREADY 拉高确认
- 有等待传输：Slave 拉低 HREADY，Master 保持地址/数据直到 HREADY 拉高
- 为什么是 2 周期：地址周期 + 数据周期，地址和数据分时复用同一组信号

### <strong>流水线（Pipelining）机制</strong>

<span class="badge-i">I</span>

- AHB 支持地址流水线：当前数据传输周期的同时，Master 可以发下一个地址
- 理想情况：连续突发时，每个周期都有数据传输（地址和数据重叠）
- 实际限制：Slave 插入等待会打断流水线，Master 必须等 HREADY 才能发新地址
- 与 AXI 对比：AXI 的 5 个通道天然流水，AHB 需要地址/数据周期严格交替

### <strong>突发传输详解</strong>

<span class="badge-i">I</span>

- SINGLE：单次传输，无突发
- INCR：未定义长度递增突发，直到 burst 结束信号
- INCR4/INCR8/INCR16：4/8/16 beat 固定长度递增
- WRAP4/WRAP8/WRAP16：回卷突发，地址到边界后回卷，用于 cache line fill
- HTRANS 信号：
  - NONSEQ：突发第一个 beat
  - SEQ：突发后续 beat
  - BUSY：Master 还没准备好发数据，插入空闲周期
  - IDLE：总线空闲

## 03-AHB仲裁与多主控制.md **[I→E]**

> <span class="badge-i">I</span> → <span class="badge-e">E</span>

### <strong>多主仲裁机制</strong>

<span class="badge-i">I</span>

- AHB 支持多 Master（如 CPU + DMA）共享总线
- 仲裁信号：HBUSREQ（请求总线）、HLOCKx（锁定总线）、HGRANT（授权）
- 仲裁策略：
  - 固定优先级：CPU > DMA > 调试，简单但可能导致低优先级饿死
  - 轮询：公平分配，但增加了切换开销
  - 混合策略：高优先级固定，同优先级轮询

### <strong>总线矩阵与地址译码</strong>

<span class="badge-i">I</span>

- AHB 总线矩阵：多主多从的交叉开关，内部通过仲裁器 + 多路选择器实现
- 地址译码：HSELx 信号，每个 Slave 对应一段地址空间
  - 例如：0x0000_0000-0x0FFF_FFFF → DDR
  - 0x2000_0000-0x2FFF_FFFF → SRAM
  - 0x4000_0000-0x4FFF_FFFF → APB 桥（外设寄存器）
- 默认 Slave：访问未映射地址时，HRESP=ERROR

### <strong>RETRY 与 SPLIT 响应 **[M]**</strong>

<span class="badge-m">M</span>

- OKAY：正常完成
- ERROR：地址错误或访问权限不足
- RETRY：Slave 忙，Master 稍后重试同一地址（不释放总线优先级）
- SPLIT：Slave 不能立即响应，让 Master 释放总线，Slave 准备好后通过仲裁器通知 Master 重试
- RETRY vs SPLIT：RETRY 占着总线重试，SPLIT 释放总线让其他 Master 用，效率更高但实现复杂
- AHB-Lite 删除：ARM 后来推出的 AHB-Lite 子集去掉了 RETRY/SPLIT/Multi-master，只保留单主简化版

## 04-AHB实战与历史演进.md **[E]**

> <span class="badge-e">E</span>

### <strong>实战 1：STM32 AHB 总线矩阵分析与 DMA 性能瓶颈</strong>

<span class="badge-e">E</span>

- 场景：STM32H7 上 DMA 从 SRAM 搬运数据到外设，实测带宽远低于理论值
- 分析：
  - STM32H7 总线矩阵：CPU（D-bus/I-bus）+ DMA1 + DMA2 + 以太网 DMA 共用 AHB 矩阵
  - DMA 与 CPU 同时访问 SRAM 时，仲裁器轮询分配，带宽减半
  - SRAM 分 Bank1/Bank2，DMA 和 CPU 访问不同 Bank 可并行
- 优化：配置 DMA 到 SRAM Bank2，CPU 用 Bank1，避免仲裁冲突

### <strong>实战 2：AHB2APB 桥接器时序与寄存器访问延迟</strong>

<span class="badge-e">E</span>

- 场景：CPU 通过 AHB 访问 APB 外设寄存器，实测延迟 3-4 周期，分析来源
- AHB2APB 桥工作原理：
  - T1：AHB 地址周期，桥采样地址并启动 APB 传输
  - T2-T3：APB 侧 SETUP + ENABLE 周期（APB 固定 2 周期）
  - T4：数据返回 AHB，HREADY 拉高
- 优化：批量访问 APB 寄存器时，用 AHB burst 减少桥接开销

### <strong>实战 3：AHB-Lite 从机 RTL 设计 **[M]**</strong>

<span class="badge-m">M</span>

- 场景：Cortex-M0 软核接自研外设，需要 AHB-Lite 从机接口
- RTL 要点：
  - 单主简化，无需仲裁信号
  - 状态机：IDLE → 地址采样 → 数据处理 → HREADY 拉高 → IDLE
  - 突发支持：可选（AHB-Lite 不要求突发），实现 INCR 突发即可
- 验证：用 ARM CMSDK AHB-Lite 验证套件测试兼容性
