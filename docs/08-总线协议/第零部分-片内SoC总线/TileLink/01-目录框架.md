## 01-TileLink基础认知与架构.md **[B→I]**

> <span class="badge-b">B</span> → <span class="badge-i">I</span>

### <strong>TileLink 核心定义与价值</strong>

<span class="badge-b">B</span>

- 什么是 TileLink
  - RISC-V 社区推出的开源片上总线协议，由 SiFive 主导设计
  - 不是 ARM 的标准，但在 RISC-V SoC 中已经成为事实标准
  - 原生支持缓存一致性、原子操作、错误报告
- 为什么 RISC-V 需要自己的总线
  - ARM 的 AMBA 需要授权，RISC-V 追求开放生态，TileLink 完全开源
  - 设计目标：简单 enough 能用手写 RTL 实现，强大 enough 支持服务器级多核
  - <span class="blue">TileLink 是 RISC-V 世界里的 AMBA，但更年轻、更开放。</span>
- 典型芯片中的 TileLink
  - SiFive U74：双核 RISC-V 通过 TileLink 连接 L2 cache、DDR 控制器、外设
  - 香山（XiangShan）处理器：中科院开源 RISC-V 高性能核，使用 TileLink 作为片内总线

### <strong>TileLink 与 AMBA 的定位差异</strong>

<span class="badge-b">B</span>

| 维度 | TileLink | AMBA |
|------|----------|------|
| 出身 | RISC-V / SiFive 开源生态 | ARM 商业生态 |
| 一致性 | 原生支持（TL-C） | 通过 ACE/CHI 扩展 |
| 协议复杂度 | 中（3 个一致性级别） | 高（AXI+ACE+CHI 多层） |
| 开源实现 | Rocket Chip、Chipyard 全开源 | ARM CoreLink 闭源，只有 spec 公开 |
| 典型用户 | RISC-V 芯片公司、学术界 | ARM SoC 厂商 |

- 不做优劣对比，只说明：ARM 生态选 AMBA，RISC-V 生态选 TileLink

### <strong>TileLink 三级一致性架构</strong>

<span class="badge-i">I</span>

- TL-UL（Uncached Lightweight）：无缓存，类似 APB，用于寄存器访问
- TL-UH（Uncached Heavyweight）：无缓存但支持原子操作和突发，类似 AHB
- TL-C（Cached Coherent）：全缓存一致性，类似 AXI+ACE
- 设计选择：外设接 TL-UL，DMA/加速器接 TL-UH，CPU 集群接 TL-C

## 02-TileLink通道与事务类型.md **[I→E]**

> <span class="badge-i">I</span> → <span class="badge-e">E</span>

### <strong>TileLink 通道架构</strong>

<span class="badge-i">I</span>

- A 通道（Acquire/Put/Get）：Master 发起请求（读/写/获取缓存行）
- B 通道（Probe）： coherence manager 向其他 Master 发窥探请求
- C 通道（Release/ProbeAck）：Master 释放缓存行或响应窥探
- D 通道（Grant/AccessAck）：coherence manager / Slave 返回响应
- E 通道（GrantAck）：Master 确认收到 Grant，完成事务闭环
- 对比 AXI：TileLink 的 5 个通道也是分离的，但语义是"一致性导向"而非"数据传输导向"

### <strong>TileLink 事务类型详解</strong>

<span class="badge-i">I</span>

- Get：读数据（类似 AXI 读）
- PutFullData / PutPartialData：全字/部分写（类似 AXI 写）
- ArithmeticData / LogicalData：原子算术/逻辑操作（RISC-V AMO 指令对应）
- Acquire：获取缓存行（读入 cache）
- Probe：窥探其他 Master 的 cache（一致性维护）
- Release：写回并释放 cache 行（cache eviction）
- Grant：授权 cache 行所有权

### <strong>TileLink 握手机制与 AXI 对比</strong>

<span class="badge-e">E</span>

- TileLink 也使用 VALID/READY 握手，但通道方向与 AXI 不同
- AXI 中读地址和读数据方向相反，TileLink 的 A 和 D 也是相反
- 关键差异：TileLink 的 B/C 通道是"反方向窥探"，这是 AXI 没有的
- 时序约束：a_valid && a_ready 时传输发生，与 AXI 相同

## 03-TileLink缓存一致性机制.md **[E]**

> <span class="badge-e">E</span>

### <strong>缓存一致性基础：为什么需要 Tile-C</strong>

<span class="badge-i">I</span>

- 多核 RISC-V，各带私有 L1 cache，同时访问同一地址
- 没有一致性：核 A 改了 cache 里的变量，核 B 读内存拿到旧值
- TileLink TL-C 的解决思路：目录式一致性（Directory-based Coherence）

### <strong>目录式一致性协议</strong>

<span class="badge-e">E</span>

- 每个 cache 行在 L2 / 目录中记录：哪些核持有、是 Shared 还是 Exclusive
- 状态机（简化）：
  - Nothing：没有核 cache 这行
  - Branch：一个或多个核只读（Shared）
  - Trunk：只有一个核可写（Exclusive）
  - Tip：某个核最新修改过，内存是旧值
- 事务流：核 A 写地址 X → 目录查状态 → 如果核 B 有副本，发 Probe → 核 B Invalidate → 目录更新 → Grant 给核 A

### <strong>TileLink 与 MESI 的关系</strong>

<span class="badge-e">E</span>

- MESI 是单级 cache 的状态机（Modified/Exclusive/Shared/Invalid）
- TileLink 是系统级一致性协议，定义了核与核之间、核与内存之间的事务交互
- 对应关系：Acquire → MESI 的过渡请求，Grant → 状态转换完成，Probe → 其他核状态变更通知

## 04-TileLink实战与RISC-V生态.md **[E]**

> <span class="badge-e">E</span>

### <strong>实战 1：Rocket Chip 生成器的 TileLink 总线结构分析</strong>

<span class="badge-e">E</span>

- 场景：用 Chipyard 配置一个双核 RISC-V SoC，分析 TileLink 总线拓扑
- 结构：
  - 两个 Rocket 核 → TL-C → L2 cache（Inclusive，目录式一致性）
  - L2 → TL-UH → DRAM 控制器
  - L2 → TL-UL → UART/SPI/GPIO 外设
- 分析：打开生成的 Verilog，数 TileLink 节点，理解 crossbar 连接关系

### <strong>实战 2：TileLink 到 AXI4 桥接器设计与时序</strong>

<span class="badge-e">E</span>

- 场景：RISC-V SoC 需要接 DDR 控制器，但 DDR IP 只有 AXI4 接口
- 桥接要点：
  - TileLink Get → AXI4 AR + R 通道
  - TileLink Put → AXI4 AW + W + B 通道
  - TileLink Acquire（缓存一致性）→ 需要额外处理：写回脏数据 + 无效化缓存
- 开源实现：Rocket Chip 自带 TileLink2AXI4 桥，可直接用

### <strong>实战 3：自定义加速器接 TileLink 接口设计 **[M]**</strong>

<span class="badge-m">M</span>

- 场景：自研 NPU 加速器需要接入 SiFive 的 TileLink 总线
- TL-UL 从机设计：
  - 状态机：IDLE → 采样地址 → 数据处理 → valid/ready 握手 → IDLE
  - 寄存器映射：控制/状态/数据 FIFO，基址由设备树分配
  - 中断输出：通过 TileLink 的独立中断线或PLIC连接
- 验证：用 Chipyard 的 TileLink DPI 仿真模型做事务级验证
