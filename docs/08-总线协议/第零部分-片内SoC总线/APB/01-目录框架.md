## 01-APB基础认知与架构.md **[B→I]**

> <span class="badge-b">B</span> → <span class="badge-i">I</span>

### <strong>APB 核心定义与价值</strong>

<span class="badge-b">B</span>

- 什么是 APB（Advanced Peripheral Bus）
  - AMBA 家族中最低速、最简化的总线标准
  - 面向"寄存器型外设"：UART、I2C、GPIO、定时器、看门狗
  - 设计哲学：简单到可以用门电路数出来，省电到可以忽略不计
- 为什么不用 AHB/APB 直接连所有外设？
  - 功耗：高速总线时钟门控困难，APB 可以单独关时钟
  - 面积：APB 从机接口只有十几个信号，AHB 需要几十个
  - 速度匹配：UART 寄存器不需要 200MHz 访问，APB 的 几十MHz 足够
  - <span class="blue">APB 是总线世界的自行车，慢但省能量，短距离最合适。</span>

### <strong>APB 与 AHB 的本质差异</strong>

<span class="badge-b">B</span>

| 特性 | APB | AHB |
|------|-----|-----|
| 通道 | 1（地址+数据复用） | 1（地址+数据复用，但支持流水线） |
| 时钟 | 可单独分频/关断 | 通常与系统同频 |
| 突发 | 不支持 | 支持 |
| 等待 | 通过 PREADY 插入 | 通过 HREADY 插入 |
| 信号数 | ~12 个 | ~20+ 个 |

- APB 不是 AHB 的"降级版"，而是"专用版"——专门为寄存器访问优化

### <strong>APB 总线信号速览</strong>

<span class="badge-i">I</span>

- PCLK / PRESETn：时钟与复位
- PADDR[31:0]：地址总线
- PWDATA / PRDATA：写数据 / 读数据（通常 8/16/32bit）
- PWRITE：读写方向
- PSELx：Slave 选择（每个 Slave 一根，由地址译码生成）
- PENABLE：传输使能（SETUP → ENABLE 状态切换）
- PREADY：Slave 就绪（插入等待周期）
- PSLVERR：传输错误（APB3/4 新增）
- PPROT[2:0] / PSTRB[3:0]：保护类型 / 写选通（APB4 新增）

## 02-APB传输时序与低功耗.md **[I]**

> <span class="badge-i">I</span>

### <strong>APB 状态机与传输时序</strong>

<span class="badge-i">I</span>

- APB 只有 3 个状态：IDLE → SETUP → ENABLE → IDLE
- 固定 2 周期传输（无等待时）：
  - SETUP 周期：PSEL 拉高、PADDR/PWRITE/PWDATA 有效
  - ENABLE 周期：PENABLE 拉高、Slave 采样并响应、PRDATA/PREADY 有效
- 有等待时：PREADY 拉低，ENABLE 状态延长，直到 PREADY 拉高
- 对比 AHB：AHB 也是 2 周期但支持流水线，APB 不支持流水线，每传输之间至少 1 个 IDLE 周期

### <strong>APB3 与 APB4 的演进</strong>

<span class="badge-i">I</span>

- APB2：最原始版本，无 PREADY/PSLVERR，Slave 必须单周期响应
- APB3：增加 PREADY（等待）和 PSLVERR（错误报告），Slave 可以慢响应
- APB4：增加 PPROT（TrustZone/特权访问标记）和 PSTRB（字节选通，支持窄传输）
- 嵌入式现状：APB3 最普及，APB4 用于需要安全扩展的汽车/IoT 芯片

### <strong>低功耗设计：APB 的杀手锏</strong>

<span class="badge-i">I</span>

- 时钟门控：PCLK 可以单独关闭，整个 APB 总线和所有外设进入静态
- 外设分区分组：一组 GPIO 在 Bank A，另一组在 Bank B，可以分别开关
- 自动唤醒：中断信号从 APB 外设（如 GPIO 中断）直接连到 NVIC/ GIC，不依赖 APB 时钟
- 实际数据：APB 接口动态功耗约为 AHB 的 1/5，静态漏电约为 AHB 的 1/3

## 03-APB实战与桥接设计.md **[I→E]**

> <span class="badge-i">I</span> → <span class="badge-e">E</span>

### <strong>实战 1：AHB2APB 桥设计与寄存器访问延迟分析</strong>

<span class="badge-i">I</span>

- 场景：Cortex-M4 通过 AHB 访问 APB 外设，分析每一次寄存器读写的延迟来源
- AHB2APB 桥时序拆解：
  - T1（AHB 地址）：桥采样 HADDR，译码确定 PSELx
  - T2（APB SETUP）：桥发起 APB 传输，PSEL/PADDR/PWRITE 有效
  - T3（APB ENABLE）：PENABLE 拉高，Slave 响应，PREADY 返回
  - T4（AHB 数据）：PRDATA 传回 AHB 侧，HREADY 拉高
- 优化：如果 APB Slave 总是单周期响应（PREADY 固定拉高），桥可以省掉 1 个等待周期

### <strong>实战 2：APB 外设寄存器标准模板设计</strong>

<span class="badge-e">E</span>

- 场景：自研 UART 外设，需要一套标准的 APB 寄存器接口
- 寄存器模板：
  - 控制寄存器（CR）：使能、中断开关、模式配置
  - 状态寄存器（SR）：标志位（TXE、RXNE、ERR）
  - 数据寄存器（DR）：收发缓冲
  - 波特率寄存器（BRR）：分频系数
- APB 地址映射：每个寄存器 4 字节对齐，地址 = 基址 + 偏移
- 读写权限：控制寄存器可读写，状态寄存器只读，数据寄存器读写双功能

### <strong>实战 3：多 APB 桥层级与地址规划 **[M]**</strong>

<span class="badge-m">M</span>

- 场景：复杂 SoC 有 50+ 外设，单个 APB 桥地址空间不够
- 设计：多级 APB 桥，AHB → APB Bridge 0（0x4000_0000-0x400F_FFFF）→ APB Bridge 1（0x4010_0000-...）
- 地址规划原则：
  - 同一功能组的外设放在同一桥下（如所有通信外设：UART/SPI/I2C）
  - 考虑时钟域：桥 0 用 50MHz，桥 1 用 100MHz，桥 2 可关断
  - 预留地址空间：每个桥预留 20% 地址给未来扩展
