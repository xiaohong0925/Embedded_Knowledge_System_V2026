# 实战1：GPIO按键中断开发

> 📊 **本节难度等级：** <span class="badge-i">**I级**</span>

---

### <strong>GPIO按键中断是嵌入式Linux中断开发的“入门第一课”，也是工业控制、消费电子中最基础的交互场景（如设备复位按键、模式切换按键）。
本实战以STM32MP157开发板为载体，实现“按键按下触发中断→中断处理函数控制LED点亮，按键松开触发中断→LED熄灭”的核心功能，重点落地“<span class="red">设备树</span>中断配置”“中断注册/注销”“硬中断上下文消抖”等基础知识点，同时覆盖入门级调试技巧。</strong>


### <strong>硬件准备</strong>

| 硬件组件       | 规格/连接方式                                                                 |
|----------------|------------------------------------------------------------------------------|
| 开发板         | STM32MP157（ARM Cortex-A7双核，Linux 5.10内核）                              |
| 按键           | 一端接GPIOA_PIN0（物理引脚PA0），一端接地（下拉，按键按下时电平从高→低）       |
| LED灯          | 一端接GPIOB_PIN0（物理引脚PB0），一端通过220Ω电阻接地，GPIO输出高电平时点亮   |
| 调试工具       | 串口调试器（连接开发板UART1，波特率115200）、SD卡（烧录Linux镜像）            |

### <strong>驱动开发步骤</strong>

1. <span class="red">设备树</span>配置：添加<span class="green">GPIO</span>中断节点
<span class="red">设备树</span>是中断号解析的基础，需在STM32MP157设备树（`stm32mp157d-ev1.dts`）中添加按键设备节点，指定中断父节点、引脚、触发方式。

（1）<span class="red">设备树</span>节点编写
```dts
/* 追加到设备树根节点下 */
key_led_dev: key-led@0 {
    compatible = "stm32,key-led-irq";  // 驱动匹配关键字
    status = "okay";                  // 启用节点
    key-gpio = <&gpioa 0 GPIO_ACTIVE_LOW>;  // 按键GPIO：GPIOA_PIN0，低电平有效
    led-gpio = <&gpiob 0 GPIO_ACTIVE_HIGH>; // LED GPIO：GPIOB_PIN0，高电平有效
    interrupt-parent = <&gpioa>;      // 中断父节点：GPIOA控制器
    interrupts = <0 IRQ_TYPE_EDGE_BOTH>;  // 中断参数：引脚0，双边沿触发（按下/松开都触发）
};
```

（2）<span class="red">设备树</span>编译与烧录
<span class="red">设备树</span>修改后需重新编译并烧录到开发板，核心命令（开发主机端）：
```bash
# 进入Linux内核源码目录（已配置STM32MP157交叉编译环境）
cd linux-5.10-stm32mp
# 交叉编译设备树（交叉编译器：arm-linux-gnueabihf-）
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- stm32mp157d-ev1.dtb
# 将编译后的dtb文件拷贝到SD卡的boot分区
cp arch/arm/boot/dts/stm32mp157d-ev1.dtb /media/boot/
# 开发板重启加载新设备树
```<br>

### <strong>调试技巧：中断未触发时的排查流程</strong>

中断未触发是入门高频问题，需按“硬件→<span class="red">设备树</span>→驱动→内核”的顺序排查，以下是标准化流程（附Mermaid流程图）：


核心排查命令（开发板端）
1. 测量<span class="green">GPIO</span>电平（硬件层面）：
   ```bash
   # 安装gpio工具（STM32MP157）
   apt install gpiod
   # 读取PA0电平
   gpio get PA0
   ```
2. 读取<span class="green">GPIO</span>中断状态寄存器（硬件层面）：
   ```bash
   # STM32MP157 GPIOA_ISR寄存器地址：0x50002010
   devmem 0x50002010 32
   # 返回值二进制第0位为1，表示PA0触发中断；0表示未触发
   ```
3. 查看内核中断状态（软件层面）：
   ```bash
   # 查看irq=62的处理次数
   cat /proc/interrupts | grep 62
   # 按下按键后，处理次数应增长
   ```<br>
