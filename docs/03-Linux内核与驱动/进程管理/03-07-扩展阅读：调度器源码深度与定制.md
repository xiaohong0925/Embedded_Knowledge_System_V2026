### <strong>CFS源码核心：红黑树与vruntime机制解析
完全公平调度器（CFS）是Linux内核默认的通用调度器，
其“公平性”核心依赖红黑树（有序管理就绪任务）和虚拟运行时间（vruntime，量化任务公平性）。
嵌入式场景中CFS的定制多围绕“红黑树轻量化”“vruntime权重调整”展开，需先吃透核心源码逻辑。</strong>

1.1 红黑树核心操作：__enqueue_entity与__dequeue_entity
CFS通过红黑树（根节点为`cfs_rq->rb_root`）管理所有就绪状态的普通进程（`task_struct`），每个进程对应树中的一个节点，按`se->vruntime`（调度实体的虚拟运行时间）升序排列。
`__enqueue_entity`（入队）和`__dequeue_entity`（出队）是红黑树操作的核心函数，源码位于`kernel/sched/fair.c`。<br>

### <strong>实时调度关键机制：优先级反转解决方案源码解析
嵌入式实时场景（如电机控制、自动驾驶）的核心痛点是“优先级反转”
——低优先级进程持有高优先级进程所需的锁，导致高优先级进程阻塞。
Linux通过优先级继承（PI）和优先级天花板（PC）协议解决该问题，其中PI协议因灵活性成为主流，源码位于`kernel/sched/rt.c`。</strong>

2.1 优先级继承（PI）协议核心源码：rt_mutex机制

PI协议的核心是“当低优先级进程持有高优先级进程的锁时，临时提升低优先级进程的优先级至锁的最高申请者优先级”，实现该逻辑的核心结构是`rt_mutex`（实时互斥锁）。<br>

### <strong>嵌入式调度器定制：硬件适配与资源优化
嵌入式设备的“异构多核、资源受限、实时性差异化”特性，决定了通用调度器无法直接满足需求，需从调度域配置（核间协同）、调度类裁剪（资源优化）、异构适配（big.LITTLE）三个维度定制。</strong>

3.1 调度域（sched_domain）修改：异构多核核组划分

调度域（`sched_domain`）是内核描述CPU核组的逻辑结构，用于控制任务在核间的迁移策略。嵌入式big.LITTLE架构（如Cortex-A53/A72）需将大核与小核划分为独立调度域，避免轻负载任务运行在大核。

3.1.1 源码定制：修改sched_domain初始化逻辑

源码位于`kernel/sched/topology.c`，通过修改`build_sched_domains()`函数，将CPU0-3（A53）和CPU4-7（A72）划分为两个独立调度域：

```c


static void __init build_sched_domains(const struct cpumask *cpu_map)
{
    struct sched_domain *sd;
    struct cpumask *big_core_mask = cpumask_of(4-7);  // 大核CPU4-7
    struct cpumask *little_core_mask = cpumask_of(0-3);  // 小核CPU0-3

    // 1. 初始化小核调度域（A53）
    sd = alloc_sched_domain(little_core_mask);
    sd->flags = SD_LOAD_BALANCE | SD_BALANCE_NEWIDLE;  // 小核内部负载均衡
    sd->min_interval = 100000;  // 负载均衡最小间隔100ms
    sd->max_interval = 1000000;  // 最大间隔1s
    attach_sched_domain(little_core_mask, sd);

    // 2. 初始化大核调度域（A72）
    sd = alloc_sched_domain(big_core_mask);
    sd->flags = SD_LOAD_BALANCE | SD_BALANCE_NEWIDLE;  // 大核内部负载均衡
    sd->min_interval = 50000;  // 大核负载变化快，均衡间隔50ms
    sd->max_interval = 500000;
    attach_sched_domain(big_core_mask, sd);

    // 3. 禁止跨域迁移（大核与小核不互迁）
    sd->flags &= ~SD_CROSS_DOMAIN;
}
    
```
