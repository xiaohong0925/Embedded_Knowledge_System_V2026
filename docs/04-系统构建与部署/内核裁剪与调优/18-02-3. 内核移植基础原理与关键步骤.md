### <strong>架构适配：ARM/RISC-V/MIPS的内核差异点</strong>

架构适配是内核移植的“根基”
——Linux内核通过架构相关代码（arch目录下）适配不同CPU架构的指令集、特权级、中断模型等核心特性，移植的本质就是让内核的“架构层代码”与目标硬件的架构特性对齐。
以下从内核适配的核心维度，对比三大主流架构的关键差异，并结合内核代码结构说明适配要点：

1. 架构差异核心对比（内核视角）
| 适配维度                | ARM（v7/v8）                          | RISC-V（rv64gc）                      | 
|-------------------------|---------------------------------------|---------------------------------------|
| 特权级模型              | ARMv7：SVC/User/Abort等7种模式；ARMv8：EL0-EL3四级特权级 | U/M/S三级特权级（模块化可扩展）        | 
| 内核启动入口            | arch/arm/kernel/head.S（v7）；arch/arm64/kernel/head.S（v8） | arch/riscv/kernel/head.S              |
| 中断控制器适配          | GIC（通用中断控制器），内核通过GIC驱动框架适配（drivers/irqchip/irq-gic.c） | PLIC/CLIC（平台级/核心级中断控制器），适配代码路径drivers/irqchip/irq-plic.c 
| 页表格式与MMU初始化     | ARMv7：32位页表（支持段页式）；ARMv8：4级页表（支持52位地址） | 3级/4级页表（可配置），MMU初始化依赖csr寄存器操作 |
| 内核配置核心开关        | CONFIG_ARM/CONFIG_ARM64               | CONFIG_RISCV/CONFIG_RISCV_64          |<br>

### <strong>设备树（DTS）作用：硬件信息描述与内核解耦</strong>

设备树（Device Tree Source，DTS）是Linux内核从“硬件硬编码”走向“硬件信息可配置”的核心技术，
其核心作用是用结构化文本描述硬件拓扑和参数，让内核通过解析DTS动态适配硬件，无需修改内核代码。

在移植场景中，DTS是连接“通用内核”与“特定硬件”的桥梁。

1. DTS解决的核心问题：硬编码的弊端
在没有DTS的传统内核中，硬件信息（如串口基地址、GPIO引脚、中断号）直接写死在代码中（如板级文件board-xxx.c）。例如早期ARM平台的串口配置：
```c
// 传统硬编码方式（无DTS时）
static struct uart_port s3c2410_uart0 = {
        .mapbase    = 0x50000000,  // 串口基地址硬编码
        .irq        = IRQ_UART0,   // 中断号硬编码
        .type       = PORT_S3C2410,
};
```
这种方式的弊端是：每适配一款新硬件，就需修改板级代码并重新编译内核，兼容性极差。
而DTS通过“硬件信息与内核代码分离”，彻底解决了这一问题——移植时只需修改DTS文件，无需改动内核源码。

3. 移植中的DTS核心原则
-  最小修改原则：基于芯片厂商提供的基础DTS（如`rk3568.dtsi`），通过“DTS叠加层（Overlay）”修改板级差异（如新增外设），避免直接修改基础DTS。
-  兼容性原则：`compatible`属性需先写“板级型号”，再写“通用型号”（如`compatible = "myboard,rk3568-uart", "arm,pl011"`），确保优先匹配板级定制驱动，无定制驱动时匹配通用驱动。<br>

### <strong>板级支持包（BSP）构成：板级初始化代码+驱动集合</strong>

板级支持包（Board Support Package，BSP）是针对特定开发板/产品的“定制化内核组件集合”，
其核心目标是让通用Linux内核在“特定硬件板卡”上正常启动并工作。移植时的核心工作，就是构建或修改BSP，使其匹配目标板的硬件特性。

1. BSP的核心构成（内核目录视角）
BSP并非独立的“包文件”，而是分散在内核源码中的“架构相关代码+板级代码+驱动代码”的集合，具体构成如下：
| 构成部分                | 内核路径                          | 核心作用                                  | 移植时的关键操作                        |
|-------------------------|-----------------------------------|-------------------------------------------|-----------------------------------------|
| 板级初始化代码          | arch/<架构>/mach-<芯片>/          | 完成板级硬件的早期初始化（DRAM、时钟、串口） | 修改时钟频率、DRAM容量、串口基地址等参数 |
| 架构核心代码            | arch/<架构>/kernel/               | 实现架构级特性（特权级、中断、MMU）        | 适配架构扩展指令（如RISC-V的FPU）       |
| 板级专属驱动            | drivers/<驱动类型>/<板级驱动>/     | 适配板级专属外设（如定制按键、板载传感器）  | 编写或修改驱动，匹配DTS中的板级节点      |
| 配置文件                | arch/<架构>/configs/<板级defconfig> | 开启/关闭内核功能、驱动（如`CONFIG_SERIAL=y`） | 基于基础defconfig，裁剪冗余功能          |<br>
