    3.4.3 存储操作命令¶

    所属章节：第3章 U-Boot命令详解 > 3.4 存储命令 难度：[B] | 预计阅读时间：25分钟

    本节导读¶

    本节讲解U-Boot中最常用的三类存储操作命令：MMC/SD卡、USB存储和SPI Flash。学完本节，你将能够在U-Boot命令行中查看存储设备信息、浏览文件、从SD卡或U盘加载内核镜像到内存，以及读写SPI Flash——这些都是嵌入式开发日常调试的必备技能。

    知识点1：MMC/SD卡操作 [B] ~800字¶

    MMC（MultiMediaCard）/SD卡是嵌入式设备最常见的存储介质。U-Boot提供了一套以mmc为前缀的命令来操作这类设备。理解这些命令的关键是记住U-Boot的设备编号规则：第一个MMC设备编号为0，第二个为1，以此类推；每个设备内的分区用冒号分隔，如0:1表示第0个设备的第1个分区。

    查看MMC设备列表¶

    mmc list命令会列出当前系统识别到的所有MMC/SD卡设备：

    U-Boot> mmc list
SD/MMC: 0 (eMMC)
SD/MMC: 1 (SD)

    上面的输出表示：设备0是一块eMMC芯片（焊接在板子上），设备1是一张插入的SD卡。U-Boot启动时通常会自动初始化这些设备，但如果没有，可以用mmc dev <编号>手动切换当前操作的设备。

    查看设备详细信息¶

    切换到目标设备后，mmc info显示该存储卡的容量、总线宽度、时钟频率等关键参数：

    U-Boot> mmc dev 1      # 切换到SD卡
U-Boot> mmc info
Device: MMC
Manufacturer ID: 3
OEM: 5344
Name: SC16G
Bus Speed: 50000000
Mode: SD High Speed (50MHz)
Rd Block Len: 512
SD version 2.0
High Capacity: Yes
Capacity: 14.8 GiB

    💡 提示：如果mmc info输出异常或容量显示为0，尝试先执行mmc rescan重新扫描总线。

    查看分区表¶

    mmc part命令读取当前MMC设备上的分区表（支持MBR和GPT格式）：

    U-Boot> mmc dev 0
U-Boot> mmc part
Partition Map for MMC device 0  --   Partition Type: DOS
Part    Start Sector    Num Sectors     UUID            Type
  1     2048            524288          00000000-01     0c  # FAT32启动分区
  2     526336          30717952        00000000-02     83  # Linux ext4根分区

    上例中，eMMC有两个分区：第1分区是FAT32格式（通常存放内核和设备树），第2分区是ext4格式（根文件系统）。

    浏览与读写FAT分区文件¶

    对于FAT格式的分区，U-Boot使用fat系列命令操作文件。语法格式为fatls <接口> <设备:分区>：

    U-Boot> fatls mmc 0:1          # 列出eMMC第1分区的文件
    82308   u-boot.bin
  7811312   zImage
    40960   imx6ull-14x14-evk.dtb

3 file(s), 0 dir(s)

    fatload将文件加载到指定内存地址（加载地址通常在U-Boot环境变量中定义，如${loadaddr}）：

    U-Boot> fatload mmc 0:1 ${loadaddr} zImage
7811312 bytes read in 356 ms (20.9 MiB/s)

    fatwrite则将内存中的数据写入FAT分区，常用于更新固件：

    U-Boot> fatwrite mmc 0:1 ${loadaddr} zImage ${filesize}

    ⚠️ 陷阱：fatwrite要求目标分区已格式化为FAT16/FAT32。如果写入ext4分区会报错"Bad FAT filesystem"。

    🔴 危险：fatwrite会覆盖同名文件且不提示确认！写入前务必用fatls确认文件名正确，避免误删重要固件。

    [图1：MMC/SD卡操作流程示意]——展示从设备识别到文件加载的完整步骤

    知识点2：USB存储操作 [B] ~600字¶

    U-Boot同样支持通过USB接口操作U盘等移动存储设备。USB命令的设计逻辑与MMC命令类似，但多了一步总线初始化——因为USB是热插拔总线，U-Boot不会自动扫描所有USB端口。

    初始化USB总线¶

    U-Boot> usb start
starting USB...
USB0:   USB EHCI 1.00
scanning bus 0 for devices... 2 USB Device(s) found
       scanning bus for storage devices... 1 Storage Device(s) found

    usb start扫描USB控制器并枚举连接的设备。如果U盘在U-Boot启动后才插入，必须执行此命令才能识别。

    ⚠️ 陷阱：部分老旧U-Boot版本使用usb reset而非usb start，若一个命令无效可尝试另一个。

    查看USB存储设备¶

    U-Boot> usb storage
Device 0: Vendor: Kingston Rev: 1.00 Prod: DataTraveler
            Type: Removable Hard Disk
            Capacity: 30250.0 MB = 29.5 GB (61931520 x 512)

    输出中的"Device 0"就是后续命令中引用的设备号。如果连接了多个U盘，会依次显示Device 0、Device 1等。

    浏览和加载U盘文件¶

    USB存储设备同样使用fat系列命令访问，只需将接口名从mmc替换为usb：

    U-Boot> fatls usb 0:1
    82308   u-boot.bin
  7811312   zImage
    40960   imx6ull-14x14-evk.dtb

U-Boot> fatload usb 0:1 ${loadaddr} zImage
7811312 bytes read in 412 ms (18.1 MiB/s)

    从U盘加载文件的速度通常比SD卡稍慢，但U盘容量大、便于在PC和开发板之间传递文件，是现场调试和产线烧录的常用手段。

    💡 提示：U-Boot对U盘的FAT分区支持较好，但对NTFS或exFAT支持有限。建议将U盘格式化为FAT32以确保兼容性。

    一个完整的从U盘启动内核示例¶

    U-Boot> usb start
U-Boot> fatload usb 0:1 ${loadaddr} zImage
U-Boot> fatload usb 0:1 ${fdt_addr} imx6ull-14x14-evk.dtb
U-Boot> bootz ${loadaddr} - ${fdt_addr}

    [图2：U盘启动调试场景示意]——展示PC制作启动U盘、开发板读取并启动的流程

    知识点3：SF（SPI Flash）操作 [B] ~400字¶

    SPI Flash（Serial Flash，通常称为NOR Flash）是嵌入式系统中存放U-Boot本身、环境变量和内核的常用存储介质。与MMC/USB的块设备文件系统访问不同，SPI Flash操作更接近"原始存储"——直接按地址读写，且写入前必须先擦除。

    探测SPI Flash¶

    U-Boot> sf probe
SF: Detected w25q128bv with page size 256 Bytes, erase size 4 KiB, total 16 MiB

    sf probe自动探测并初始化SPI Flash。若系统有多个SPI控制器，可用sf probe <总线>:<片选>指定。

    读取数据到内存¶

    U-Boot> sf read ${loadaddr} 0x100000 0x500000

    语法为sf read <内存地址> <Flash偏移地址> <长度>。上例从Flash地址0x100000（1MB处）读取5MB数据到内存。

    擦除与写入¶

    U-Boot> sf erase 0x100000 0x500000      # 擦除5MB区域
U-Boot> sf write ${loadaddr} 0x100000 0x500000  # 从内存写入5MB到Flash

    🔴 危险：sf erase会不可逆地擦除Flash数据！执行前务必三次确认：地址正确、长度正确、区域不是U-Boot自身所在区域。擦除U-Boot所在区域会导致设备变砖。

    ⚠️ 陷阱：SPI Flash写入只能将1变为0，不能将0变为1，因此写入前必须擦除。直接写入未擦除的区域会导致数据混乱。

    💡 提示：很多开发板的Flash分区布局在U-Boot源码或文档中有明确定义。执行写操作前，用printenv查看mtdparts环境变量，了解内核、设备树、根文件系统的固定存放位置。

    [图3：SPI Flash分区布局示意]——展示典型嵌入式设备的16MB SPI Flash分区规划

    本节总结¶

    概念 要点 常用操作
    MMC/SD卡 设备编号从0开始，分区用设备:分区表示 mmc list→mmc dev N→fatls mmc 0:1→fatload
    USB存储 需先usb start初始化总线 usb start→usb storage→fatls usb 0:1→fatload
    SPI Flash 按物理地址访问，写前必须擦除 sf probe→sf read→sf erase→sf write
    文件加载 fatload将FAT分区文件加载到内存地址 fatload mmc/usb 设备:分区 ${loadaddr} 文件名
    危险操作 fatwrite和sf erase/write会覆盖原有数据 执行前用fatls/sf read确认目标区域内容
